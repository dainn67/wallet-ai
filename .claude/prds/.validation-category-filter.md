## Recommendations
- **FR-3 edit navigation gap:** The scenario says "on save, the popup reflects updated values when it re-renders" but doesn't specify the navigation behavior — does the edit sheet open on top of the popup (popup stays open underneath) or does it replace it? Clarify to avoid a guess during implementation.
- **NTH-1 trigger condition:** NTH-1 defers empty-state as rare, but a parent category with all-sub records AND the user tapping the parent will always show at least the sub-groups, never an empty popup. The only real empty-state case is tapping a sub with 0 records. Consider tightening the NTH-1 scope to sub-category-only taps.
- **US-3 persona re-use:** US-3 is assigned to "Specialist" like US-1, but the union-view behavior actually aligns more with the Organizer persona (wanting to see the full picture at a glance). Not a blocking issue, but worth considering for story prioritization.
