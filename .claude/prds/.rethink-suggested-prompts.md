# Product Brief: suggested-prompts

## Vấn đề (Đã Reframe)

User quay lại Wally AI mỗi ngày để ghi cùng một khoản chi tiêu quen thuộc (cà phê, cơm trưa, grab). Server đã phân tích pattern và sinh ra `suggestedPrompts` trong greeting JSON, nhưng client im lặng bỏ qua dữ liệu này — user vẫn phải gõ tay từng ký tự như lần đầu tiên. Cần pipe data đó lên UI dưới dạng flow 3 bước có thể chỉnh sửa để giảm ma sát ghi chép lặp lại mà không mất đi sự kiểm soát của user.

## Yêu cầu Gốc vs Hướng đi Mới

- **Gốc:** Parse `suggestedPrompts` JSON, hiển thị chip ngang trên input, tap prompt → fill input → tap action → append → send → xóa prompt khỏi list.
- **Reframe:** Giữ nguyên flow 3 bước. Không rút gọn thành one-tap hay flat chip vì amount thay đổi liên tục — user cần xem prompt trước, chọn action gần đúng, rồi chỉnh sửa text tự do trước khi gửi.
- **Insight chính:** Flow 3 bước không phải friction — đó là intentional control. One-tap sẽ tệ hơn vì user phải xóa số cũ để gõ số mới, mất nhiều thao tác hơn.

## User Mục tiêu

**Chính:** Returning user có stored pattern — người mở app 1-2 lần/ngày để ghi chi tiêu quen thuộc, muốn làm nhanh nhưng cần chỉnh giá trước khi confirm.

## Kết quả Mong muốn

User mở app, thấy chip "Bánh mì" ngay phía trên input, tap một cái, thấy "15k / 20k", chọn "20k" (hoặc xóa gõ "18k"), nhấn send — xong trong 3-4 giây thay vì 10 giây gõ. Còn lại chip "Cà phê" nếu chưa dùng. Cảm giác: app biết mình, không cần dạy lại mỗi ngày.

## Quyết định Scope

**Mode:** HOLD — scope gốc đã đúng, không mở rộng, không cắt bớt.

- ✅ TRONG:
  - Parse `{"suggestedPrompts": [...]}` object từ greeting JSON (branch riêng, không đụng record parsing)
  - `SuggestedPrompt` model: `prompt` (String) + `actions` (List\<String\>)
  - `ChatProvider` fields: `_suggestedPrompts`, `_activePromptIndex` (nullable), `_showingActions`
  - Chip bar widget (`SuggestedPromptsBar`) — horizontal scroll, ẩn khi list rỗng
  - Tap prompt → clear input, set text = prompt, show action chips
  - Tap action → append " {action}" vào input, hide action chips
  - Send with active prompt → remove that prompt from list
  - Handle `actions: []` (no action row, input pre-filled chỉ)
- ❌ NGOÀI:
  - Flat chip (prompt + amount gộp) — user cần sửa amount, gộp lại gây friction
  - One-tap record (bypass chat) — không phù hợp flow hiện tại
  - Prompts từ non-greeting messages — chỉ greeting có pattern context
  - Persist prompts qua app restart — session-only, đơn giản hơn
  - Dismiss button (NTH) — defer, user có thể ignore chip bar

## Bối cảnh Sản phẩm

- `ai-pattern-analyze` (complete) là upstream: sinh `long_term_user_pattern` → greeting dùng để tạo `suggestedPrompts` → tính năng này consume
- `update-message-body` (complete) đã thiết lập delimiter parsing (`--//--`) và record JSON branch — cần thêm branch `suggestedPrompts` song song, không đụng branch cũ
- `record-provider` (backlog) không block và không conflict — parallel safe
- Không có UI component nào tương tự trong codebase hiện tại → greenfield widget

## Quyết định Đã đưa ra

| # | Quyết định | Lựa chọn | Lý do |

## Đặt cược & Rủi ro Chính

**Đặt cược:** User có chi tiêu lặp lại đủ nhất quán để server sinh `suggestedPrompts` có ý nghĩa.
**Nếu sai:** Chip bar hiện gợi ý vô nghĩa, gây distraction thay vì delight — user ignore hoặc frustrated.
**Giảm thiểu:** Server đã có logic kiểm tra pattern quality trước khi sinh prompts; nếu pattern yếu, không sinh JSON → chip bar không xuất hiện. Feature tự gracefully absent.

## Quy mô PRD Đề xuất

**Medium** — 4-5 files, 5 functional requirements rõ ràng, không có migration, không có external dependency mới.

## Câu hỏi Mở cho PRD Discovery

1. Khi user tap action rồi chỉnh sửa text thêm (ví dụ "Bánh mì 20k extra"), action chips có nên ẩn ngay khi tap action không, hay ẩn khi user bắt đầu gõ?
2. Nếu user send mà KHÔNG tap bất kỳ prompt nào (free text), toàn bộ prompt list giữ nguyên — confirm?
3. Sau khi xóa hết prompt, chip bar ẩn — nếu user xóa hết input và muốn dùng lại prompt đã dùng, có support không? (Khuyến nghị: không, session-use-once)
