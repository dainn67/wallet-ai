# T1: Theme & token foundation

## Context
Every subsequent task in this epic depends on a single source of truth for all visual tokens. This task creates `AppTheme.light()` and the supporting constant classes, then wires it into `MaterialApp`. Without this, all tab/popup tasks would have to hardcode values — exactly what NFR-1 forbids.

## Description
Create `lib/configs/app_theme.dart` with all design tokens (colors, spacing, radii, elevation, typography) and a `ThemeExtension<AppSemanticColors>` for income/expense/transfer/category tokens not covered by Material 3's `ColorScheme`. Wire `MaterialApp(theme: AppTheme.light())` in `lib/main.dart`. No UI surfaces change in this task — only the token layer is built.

Architecture decisions in scope: AD-1 (centralized `AppTheme`), AD-2 (`ThemeExtension<AppSemanticColors>`).

## Acceptance Criteria
- [ ] **FR-1 / Token existence:** `AppColors.primary` equals `Color(0xFF8B5CF6)`; `AppColors.tertiary` equals `Color(0xFFEC4899)`; `AppSemanticColors.incomeGreen` equals `Color(0xFF22C172)`; `AppSemanticColors.expenseRed` equals `Color(0xFFEF4444)`.
- [ ] **FR-1 / Extension registration:** `Theme.of(context).extension<AppSemanticColors>()` returns non-null in any widget under `MaterialApp`.
- [ ] **FR-1 / Component themes:** `AppBar`, `NavigationBar`, `Card`, `Chip`, `FilledButton`, `OutlinedButton`, `TextButton`, `InputDecoration`, `Dialog`, `BottomSheet` all have explicit component theme overrides — none left as stock Material 3 defaults.
- [ ] **FR-1 / Spacing & radius constants:** `AppSpacing.lg == 16`, `AppRadius.card == 16`, `AppRadius.pill == 999` are accessible from any file that imports `app_theme.dart`.
- [ ] **NFR-2 / No test regressions:** `fvm flutter test` passes with zero failures after this task.

**`AppSemanticColors extends ThemeExtension<AppSemanticColors>`**:
```dart

final Color expenseRed;    // Color(0xFFEF4444)

final List<Color> categoryAccents; // 6 colors: violet, blue, orange, pink, emerald, slate
```
Implement required `copyWith` and `lerp` overrides. `lerp` interpolates each field using `Color.lerp`.

    margin: EdgeInsets.zero,

    shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(AppRadius.chip)),

    border: OutlineInputBorder(borderRadius: BorderRadius.circular(AppRadius.input), borderSide: BorderSide(color: AppColors.outline)),

    foregroundColor: AppColors.onPrimary,

    padding: EdgeInsets.symmetric(horizontal: AppSpacing.xl, vertical: AppSpacing.md),

  outlinedButtonTheme: OutlinedButtonThemeData(style: OutlinedButton.styleFrom(
    foregroundColor: AppColors.primary,

## Technical Details
- **Approach:** AD-1 (centralized config) + AD-2 (`ThemeExtension`). Single file, all tokens inline.
- **Pattern to follow:** Existing configs follow the same structure — see `lib/configs/app_config.dart` for the class-with-static-fields pattern.
- **Critical edge case:** `AppSemanticColors.lerp` must handle null `other` — return `this` if `other` is null. This is the standard `ThemeExtension` contract.
- **No google_fonts import** — `fontFamily` is a string literal `'PlusJakartaSans'`; the font is resolved from `pubspec.yaml` assets (T2).
