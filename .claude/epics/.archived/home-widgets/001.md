## Context

The widget rewrite needs a stable spec before any code is written: per-breakpoint layout matrix, confirmed camera-icon source, and a known call site for the existing ChatTab camera picker. This task is foundational — T2/T3/T4 all depend on its output.

## Description

Three concrete outputs, no production code change:

1. Write the per-breakpoint layout matrix as a table inside `docs/features/home-widget.md` (create the file if missing).
2. Verify `android.R.drawable.ic_menu_camera` renders correctly on stock Android emulator (API 26, 30, 34). If any render shows a placeholder/missing glyph, fall back to a bundled vector asset in `android/app/src/main/res/drawable/`.
3. Locate the current ChatTab code path that opens the system camera picker — record the exact file path and line range so T4 can refactor it without searching.

## Acceptance Criteria

- [ ] **FR-1 / Layout matrix:** `docs/features/home-widget.md` contains a markdown table listing all 5 breakpoints (1×1, 1×2, 2×1, 2×2, 3×2+) and which elements each shows (bar, label, write icon, camera icon, stats).
- [ ] **FR-3 / Camera icon source:** A decision is recorded in the doc: either "use `android.R.drawable.ic_menu_camera`" or "bundle `ic_camera.xml` vector at `android/app/src/main/res/drawable/ic_camera.xml`".
- [ ] **T4 hand-off:** The doc records the exact file + line range of the current ChatTab camera-picker call (e.g. `lib/screens/home/tabs/chat_tab.dart:142-168`).

## Implementation Steps

### Step 1: Audit existing ChatTab camera trigger
- Open `lib/screens/home/tabs/chat_tab.dart`
- Find the handler that calls `image_picker` with `ImageSource.camera` (likely named `_pickFromCamera`, `_openCamera`, or similar)
- Note: exact function name, line range, what permissions it requests, and what it does with the resulting `XFile`
- Record the findings as a code-block snippet in the doc

### Step 2: Verify camera drawable
- Render a minimal Glance widget on an API 26 emulator using `Image(provider = ImageProvider(android.R.drawable.ic_menu_camera))`. Note: this can be a 5-minute scratch build, not a real PR.
- If the icon renders cleanly on API 26, 30, 34 — record "use system drawable."
- If any API/emulator shows a placeholder — create `android/app/src/main/res/drawable/ic_camera.xml` (a Material camera vector) and record "use bundled vector."

### Step 3: Write layout matrix
- Create or update `docs/features/home-widget.md` with a section titled "Layout breakpoints"
- Include a markdown table with columns: Breakpoint, Size (dp), Bar, Label, Write icon, Camera icon, Stats
- Fill in per the epic's §Layout spec table
