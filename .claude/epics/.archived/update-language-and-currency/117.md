# Task: Implement protected currency change flow

## Context
Changing the primary currency is a destructive action because historical records in a different currency would lead to invalid balance calculations.

## Description
Update the currency selector in the Drawer to trigger a `ConfirmationDialog`. If confirmed, call `RecordProvider.resetAllData()` and then update the `AppCurrency` in `LocaleProvider`.

## Acceptance Criteria
- [ ] **FR-4 / Warning:** Selecting a new currency opens the destructive `ConfirmationDialog`.
- [ ] **FR-4 / Reset:** Confirming deletes all records and resets balances to 0 before changing the currency symbol.
- [ ] **NFR-2 / Integrity:** The currency does not change if the user cancels or the reset fails.

## Implementation Steps
1. Add a currency selector in `HomeScreen` drawer.
2. In the `onChanged` handler:
   - If selected currency is different from current:
   - Show `ConfirmationDialog` with warning: "Changing currency will delete all your records to maintain data consistency. Are you sure?"
3. On `onConfirm`:
   - Call `context.read<RecordProvider>().resetAllData()`.
   - Call `context.read<LocaleProvider>().setCurrency(newCurrency)`.
   - Close drawer.

## Tests to Write
- Integration test: Verify the "reset then change" sequence.
