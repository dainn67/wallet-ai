# Task: Parse suggested_category in ChatProvider stream handler

## Context

The server already returns a `suggested_category` object per record in the JSON response (e.g., `{"name": "Streaming", "type": "expense", "parent_id": -1, "message": "..."}`). The Flutter client currently ignores this field — it is never extracted from the JSON and never reaches the UI. This task adds the extraction: a one-liner inside the existing record-parsing loop in `ChatProvider._handleStream` that passes the parsed suggestion to the `Record` constructor.

## Description

In `ChatProvider._handleStream`'s `onDone` callback, there is a `for` loop that iterates over the parsed JSON records array (around line 195 of `chat_provider.dart`). For each item, extract `item['suggested_category']` and call `SuggestedCategory.fromJson(...)` on it — this returns `null` silently for any absent or malformed value. Pass the result to `Record(..., suggestedCategory: parsedSuggestion)`. The record is still saved to DB via `createRecord` with `categoryId: -1`; the `suggestedCategory` field is transient and NOT written to SQLite.

This is an additive change to a single method — existing records without `suggested_category` continue to work identically.

## Interface Contract

### Receives from T001:

  - Function: `SuggestedCategory.fromJson(dynamic json)` → `SuggestedCategory?`
  - Guaranteed: never throws; returns `null` for null/malformed/absent input
- File: `lib/models/record.dart`
  - Constructor param: `suggestedCategory: SuggestedCategory?`
  - Guaranteed: field is ignored by `toMap()` / `fromMap()`, so DB write is unchanged

- [ ] **FR-1 / Normal record unaffected:** When a record JSON item has a valid `category_id` and `suggested_category: null`, the resulting `Record` has `suggestedCategory == null` and `categoryId` set correctly — behavior is identical to before this change.

## Technical Details

- **Approach:** Additive one-liner inside an existing loop. No structural changes to `ChatProvider`.
- **Files to modify:** `lib/providers/chat_provider.dart` only — single method, single addition.
- **Patterns to follow:** Existing field extractions in the same loop (e.g., `item['amount']`, `item['category_id']`). Same style.
- **Edge cases:**
  - `item['suggested_category']` may not be a Map but could be null, a string, or an array. `SuggestedCategory.fromJson(dynamic)` handles all of these — no guard needed at the call site.
  - `category_id` may be `"-1"` (string) even when `suggested_category` is present — this is the expected server behavior. The existing parsing for `categoryId` is unaffected.
