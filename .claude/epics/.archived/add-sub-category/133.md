# Task: Categories Tab UI Refactor

## Context
Refactor the Categories tab to use `ExpansionTile` for parent categories, displaying their sub-categories and an "Add Sub-category" button within the tile.

## Interface Contract
- **Receives from T132**: Structured `Map<int, List<Category>>` of sub-categories from `RecordProvider`.

## Acceptance Criteria
- [ ] Parent categories (`parentId == -1`) are displayed as top-level `ExpansionTile` widgets.
- [ ] Expanding a tile reveals all its sub-categories.
- [ ] The last item in each expanded list is an "Add Sub-category" button.
- [ ] Clicking "Add Sub-category" correctly prompts for a name and saves with the current `parentId`.

## Implementation Steps
1.  **Modify `lib/screens/home/tabs/categories_tab.dart`**:
    *   Update `build` method.
    *   Fetch only parent categories (`parentId == -1`) for the top-level list.
    *   For each parent, return an `ExpansionTile`.
    *   In the `children` property of the `ExpansionTile`, map the sub-categories from `RecordProvider.getSubCategories(parentId)`.
    *   Append a button widget as the final item in the `children` list for "Add Sub-category".
    *   Implement the logic to show a dialog for the new sub-category name and call `RecordProvider.addCategory`.

## Verification
- Verify that tapping a parent category expands the tile correctly.
- Verify that a sub-category can be added to a parent.
- Verify that sub-categories are correctly displayed under their respective parents.
