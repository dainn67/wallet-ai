# Task: Update Category Model

## Context
To support sub-categories, the `Category` model needs a self-reference through a `parentId` field. This is the foundational step for the hierarchical category system.

## Acceptance Criteria
- [ ] `Category` model includes `final int parentId;`
- [ ] `parentId` defaults to -1 (indicating a parent category) if not provided.
- [ ] `toMap()` and `fromMap()` correctly handle `parent_id`.
- [ ] `copyWith()` includes `parentId`.
- [ ] Model remains immutable.

## Implementation Steps
1.  **Modify `lib/models/category.dart`**:
    *   Add `parentId` field to the class.
    *   Update constructor to include `this.parentId = -1`.
    *   Update `toMap` to include `'parent_id': parentId`.
    *   Update `fromMap` to read `'parent_id'` from the map (default to -1).
    *   Update `copyWith` and `toString`.

## Verification
- Run `fvm flutter test test/models/category_test.dart` (create if missing).
- Verify that a `Category` object can be created with and without a `parentId`.
