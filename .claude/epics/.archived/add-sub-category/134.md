# Task: UI Hierarchical Name Display

## Context
Ensure all UI components that display a category name show the full "Parent - Sub" format correctly for improved clarity.

## Interface Contract
- **Receives from T132**: Correct "Parent - Sub" string from `RecordProvider.getCategoryName`.

## Acceptance Criteria
- [ ] Transaction cards (`RecordWidget`) display the full category string.
- [ ] Record edit screen (`EditRecordPopup`) displays the full category string.
- [ ] Categories are truncated or wrapped gracefully if the string is too long.

## Implementation Steps
1.  **Modify `lib/components/record_widget.dart`**:
    *   Ensure the `Text` widget displaying the category name uses `context.read<RecordProvider>().getCategoryName(record.categoryId)`.
    *   Set `overflow: TextOverflow.ellipsis` and `maxLines: 1`.
2.  **Modify `lib/components/popups/edit_record_popup.dart`**:
    *   Update the category selection display or text to show the full name returned by the provider.

## Verification
- Verify that records in the transaction list show the "Parent - Sub" format.
- Verify that the edit popup shows the correct full name for the selected category.
