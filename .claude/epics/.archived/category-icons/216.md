# Task: Integration verification & cleanup

## Context
Final quality gate before marking the category-icons epic complete. Ensures all prior tasks integrate correctly end-to-end, all PRD MUST requirements are demonstrably met, all success criteria thresholds are validated, and nothing in the rest of the app regressed. This is the only place the success criteria from the PRD's _Success Criteria_ table get verified holistically — individual tasks verify their own slice, this task verifies the whole.

## Description
Run the full test suite, validate the four PRD success criteria, confirm zero `emoji IS NULL OR emoji = ''` rows in the DB, and walk the four PRD user stories end-to-end on a real device. Verify the living docs are updated. Confirm the epic file's `progress` and `status` reflect completion.

## Acceptance Criteria

- [ ] **All upstream tasks closed:** Tasks 211, 212, 213, 214, 215 are all `status: done` in their frontmatter and on GitHub.
- [ ] **Full test suite green:** `fvm flutter test` runs to completion with no failures, no skipped tests beyond pre-existing ones.
- [ ] **No build regression:** `fvm flutter build apk --debug` succeeds with no errors.
- [ ] **No analyzer regression:** `fvm flutter analyze` reports no new warnings or errors introduced by this epic.
- [ ] **FR-1, FR-6 / Success Criterion 1 (all categories have emoji):** Run `SELECT COUNT(*) FROM Category WHERE emoji IS NULL OR emoji = ''` against (a) a fresh-install DB and (b) a v9-upgraded DB. Both return `0`.
- [ ] **FR-2, FR-3, NFR-1 / End-to-end suggestion flow:** Walk through US-1 on a real device against the deployed-to-dev server (task 212): trigger a chat message that produces a new-category suggestion, observe the emoji in the banner message, tap Confirm, verify the new Category row in Categories tab has that emoji.
- [ ] **FR-4 / End-to-end edit flow:** Walk through US-2: open a category, change the emoji via the OS emoji keyboard, save, verify it persists across app restart.
- [ ] **FR-5 / End-to-end render parity:** Walk through US-4: verify the same category's emoji renders in all three sites (Categories tab list, record card subtitle, suggestion banner).
- [ ] **NFR-2 / Success Criterion 4 (≤5ms regression):** Confirmed by task 215's `tests/integration/epic_category_icons/perf-notes.md` — review file.
- [ ] **NFR-3 / Success Criterion 3 (no missing-glyph):** Confirmed by task 215's screenshots on iOS + Android — spot-check.
- [ ] **Living Docs:** `docs/features/category-icons.md` exists and accurately reflects shipped behavior; `project_context/architecture.md` mentions `Category.emoji` + v9→v10 migration.
- [ ] **No NICE-TO-HAVE bleed-through:** NTH-1 (AI-suggests-on-rename) is NOT implemented in this epic — confirm by grep.
- [ ] **Epic frontmatter:** `.claude/epics/category-icons/epic.md` frontmatter has `status: completed` and `progress: 100%`.

## Implementation Steps

### Step 1: Confirm upstream task completion
- For each of 211, 212, 213, 214, 215, verify:
  - The corresponding task file's frontmatter has `status: done`.
  - The GitHub issue is closed.
  - The task's Verification Checklist items are all checked.

### Step 2: Run the full client suite
- `fvm flutter analyze` — must report 0 new issues vs main.
- `fvm flutter test` — all tests must pass.
- `fvm flutter build apk --debug` — must succeed.

### Step 3: Run the server suite (delegated check)
- In `../chatbot-flow-server/`, run the wallyai test suite (per server repo conventions).
- Confirm task 212's new tests pass and existing wallyai tests have no regressions.

### Step 4: Validate Success Criterion 1 (DB invariant)
- Fresh install:
  - Delete app data, run `fvm flutter run`, wait for `RecordRepository.init()` to complete.
  - Use the SQLite debug query path (test_tab or a one-off `flutter test` script) to execute `SELECT COUNT(*) FROM Category WHERE emoji IS NULL OR emoji = ''`.
  - Expect `0`.
- Upgrade install:
  - Restore a v9 snapshot of `data.db` (or build a snapshot from a previous main checkout), launch this build, wait for migration to finish, run the same query.
  - Expect `0`.

### Step 5: End-to-end walkthroughs on a real device
- **US-1 (AI suggests emoji):** Send a message like "I bought a bus ticket 50k from cash". Confirm:
  - Suggestion banner appears with an emoji visible inline in the message text.
  - Tap Confirm.
  - Open Categories tab — new category appears with the suggested emoji.
- **US-2 (edit emoji):** Open the new category's edit dialog. Change the emoji via OS keyboard. Save. Reopen — emoji persists. Restart the app — emoji still persists.
- **US-3 (upgrade visual):** (Already validated in Step 4 via the v9 snapshot.) Confirm by opening Categories tab on the upgraded build: every row has a non-empty emoji.
- **US-4 (consistent rendering):** For the same category, confirm the emoji appears in:
  - Categories tab list (leading element)
  - A record card subtitle on Records tab (prefixed to name)
  - A suggestion banner (if another suggestion fires for this category — easier to verify by triggering a chat that proposes a category with similar name)

### Step 6: Living Docs review
- Open `docs/features/category-icons.md` — verify it describes the shipped behavior (no stale references to deferred NTH-1 features).
- Open `project_context/architecture.md` — verify `Category.emoji`, v9→v10 migration, and render sites are mentioned.

### Step 7: Close the epic
- Update `.claude/epics/category-icons/epic.md` frontmatter:
  - `status: completed`
  - `progress: 100%`
  - `updated: <current ISO datetime>`
- Optionally append a `## Outcome` section to the epic with one-paragraph summary (what shipped, what was deferred).
- Mark this task `status: done` and close its GitHub issue.
