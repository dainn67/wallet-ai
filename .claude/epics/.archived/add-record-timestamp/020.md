# Task: Implement Month Grouping in RecordsTab

## Context
Grouping records by month in the list provides better structure and helps users find transactions from a specific time period.

## Description
Modify the `RecordsTab` to group the list of records by month and year. This involves iterating through the sorted records and inserting `MonthDivider` widgets whenever a new month/year is encountered. The implementation should be efficient to avoid performance issues in longer lists.

## Acceptance Criteria
- [ ] **FR-2 / Grouping:** Records are visually separated by month headers (e.g., "March 2026").
- [ ] **FR-2 / Sorting:** Records within each month remain in their correct sorted order (most recent first).
- [ ] **NFR-1 / Performance:** Scrolling remains smooth with no stutter in lists up to 500 items.

## Implementation Steps

### Step 1: Create Grouping Helper Method
- Open `lib/screens/home/tabs/records_tab.dart`.
- Implement a method `List<Widget> _buildGroupedRecords(List<Record> records)`:
  - Iterate through the records.
  - Track the "current month" being processed.
  - If a record's month/year (from `createdAt`) differs from the current, insert a `MonthDivider`.
  - Add the `RecordWidget` for each record.
  - Return the combined list of widgets.

### Step 2: Update RecordsTab build method
- Replace the direct `...records.map(...)` logic with a call to `_buildGroupedRecords(records)`.
- Ensure appropriate spacing around dividers.
