# Task: Implement Redesigned RecordsOverview

## Context
The current records overview box in `RecordsTab` is functional but lacks a modern, polished design. This task implements a redesigned overview that includes a horizontal source list and clearer summarized statistics.

## Description
Create a new `RecordsOverview` component in `lib/components/records_overview.dart`. It should feature a horizontal list of `MoneySource` objects and display total income, expense, and balance. Replace the existing overview in `RecordsTab`.

## Implementation Steps
1. **Create `lib/components/records_overview.dart`**:
   - Define a `StatelessWidget` called `RecordsOverview`.
   - Add parameters for `totalBalance`, `totalIncome`, `totalExpense`, and `List<MoneySource> sources`.
   - Implement the **Horizontal Source List**:
     - Use `ListView.builder` with `scrollDirection: Axis.horizontal`.
     - Each source item should be a Material 3 `Card` showing the source name and total.
   - Implement the **Summary Stats Section**:
     - Below the source list, display the "Total Income" and "Total Expense" prominently.
     - Use the project's established color scheme for income (green) and expense (red/orange).
   - Use `GoogleFonts.poppins` throughout.
2. **Update `lib/components/components.dart`**:
   - Export `records_overview.dart`.
3. **Update `lib/screens/home/tabs/records_tab.dart`**:
   - Replace the old `_buildOverviewCard` logic with the `RecordsOverview` component.
   - Pass the relevant data from the `RecordProvider` to the component.

## Acceptance Criteria
- [ ] **FR-2 / Sources:** Displays a horizontal, scrollable list of money sources with their names and amounts.
- [ ] **FR-3 / Summary:** Correctly shows total income and total expense based on the current list of records.
- [ ] **US-2 / Clarity:** Total balance is prominently displayed.
- [ ] **NFR-1 / Styling:** Adheres to Material 3 design and uses `GoogleFonts.poppins`.

## Technical Details
- Use `SizedBox` with a fixed height (e.g., 80-100) to constrain the horizontal `ListView`.
- Ensure the overview card has a modern background (e.g., subtle gradient or Material 3 container style).

## Verification
1. **Manual Check:** Navigate to the Records tab and verify the new overview UI.
2. **Scrolling Check:** Test horizontal scrolling with 3+ money sources.
3. **Data Check:** Verify the income, expense, and balance totals match the expected data in the provider.
