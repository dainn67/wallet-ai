# Task: Create RecordWidget Component

## Context
Duplicate UI code for record items (income/expense) exists in `ChatTab` and `RecordsTab`. This task extracts that logic into a reusable Material 3 component to ensure visual consistency and better maintainability.

## Description
Create `lib/components/record_widget.dart` as a stateless "dumb" widget. It must receive a `Record` model via its constructor. The UI should be a refined version of the existing `_buildRecordCard` from `ChatTab`, using Material 3 principles (proper elevation, spacing, and typography).

## Implementation Steps
1. **Create `lib/components/` directory** if it doesn't exist.
2. **Implement `RecordWidget` in `lib/components/record_widget.dart`**:
   - Use `StatelessWidget`.
   - Take `final Record record` in the constructor.
   - Use `GoogleFonts.poppins` for all text styles.
   - Use `Theme.of(context).colorScheme` for colors.
   - Distinguish income vs expense using icons and colors (Green for income, Red for expense).
   - Ensure the description uses `TextOverflow.ellipsis` and is wrapped in `Flexible` or `Expanded` to prevent layout overflows.
3. **Create/Update `lib/components/components.dart`**:
   - Export `record_widget.dart` for easy access.

## Acceptance Criteria
- [ ] **FR-1 / Happy path:** `RecordWidget` renders correctly when passed a valid `Record` model.
- [ ] **FR-1 / Data display:** Shows description, amount, currency, and type-specific icons/colors.
- [ ] **NFR-1 / Visuals:** Uses `GoogleFonts.poppins` and Material 3 design patterns.
- [ ] **NFR-2 / Robustness:** Handles long descriptions without overflowing its container.

## Technical Details
- Follow the existing style found in `lib/screens/home/tabs/chat_tab.dart` (line 155, `_buildRecordCard`).
- Use `isExpense = record.type == 'expense'` for conditional styling.

2. **Visual Audit:** Verify colors match the app's established theme.
