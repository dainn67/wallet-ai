# Task: Database Refactor and Update Logic

## Context
The database schema needs modernization and `updateRecord` needs atomic delta logic.

## Description
Rename the table to `Record`, change the column name to `last_updated`, and implement transactional logic for record updates that correctly handles source balance changes.

## Implementation Steps
1. Open `lib/repositories/record_repository.dart`.
2. Increment `_dbVersion` to 6.
3. Update `_onCreate`:
    - Rename table `record` to `Record`.
    - Rename column `created_at` to `last_updated`.
4. Update `_onUpgrade`: ensure it handles the move to version 6 (fresh start is acceptable as per request).
5. Update all methods (`createRecord`, `getAllRecords`, `getRecordById`, `deleteRecord`) to use the new table and column names.
6. Refactor `updateRecord`:
    - Use `database.transaction`.
    - Fetch the old record state.
    - Reverse the financial impact of the old record on its original source.
    - Apply the financial impact of the new record on its (possibly new) source.
    - Update the row in the `Record` table.

## Acceptance Criteria
- [x] Database version is 6.
- [x] Table `Record` exists with column `last_updated`.
- [x] `updateRecord` correctly adjusts `MoneySource` balances when amount or source changes.
- [x] All repository tests pass (or are updated to pass).
