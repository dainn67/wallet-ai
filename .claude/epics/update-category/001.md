# Task: ToastService & main.dart Integration

## Context
Người dùng cần nhận được thông báo phản hồi (thành công, lỗi, cảnh báo) khi thực hiện các thao tác CRUD. Task này thiết lập một hệ thống thông báo toàn cục (Toast) sử dụng Pattern Singleton để có thể gọi từ bất kỳ đâu trong app.

## Description
Triển khai `ToastService` sử dụng `GlobalKey<ScaffoldMessengerState>`. Đăng ký key này trong `MaterialApp` tại `main.dart` để cho phép hiển thị SnackBar/Toast mà không cần truyền `BuildContext`.

## Acceptance Criteria
- [ ] **FR-4 / Singleton Access:** `ToastService` có thể được truy cập thông qua `ToastService()` (Singleton).
- [ ] **FR-4 / Global Key Integration:** `MaterialApp` sử dụng `scaffoldMessengerKey` từ `ToastService`.
- [ ] **FR-4 / Notification Types:** Hỗ trợ 3 phương thức: `showSuccess`, `showError`, `showWarning` với màu sắc tương ứng (Xanh, Đỏ, Vàng/Cam).
- [ ] **FR-4 / Auto-hide:** Thông báo tự động biến mất sau 2-3 giây.

## Implementation Steps

### Step 1: Create ToastService singleton
- Create `lib/services/toast_service.dart`
- Define class `ToastService` with a private constructor and a factory singleton.
- Add `static final GlobalKey<ScaffoldMessengerState> messengerKey = GlobalKey<ScaffoldMessengerState>();`.
- Implement methods:
  - `showSuccess(String message)`: Displays SnackBar with green background.
  - `showError(String message)`: Displays SnackBar with red background.
  - `showWarning(String message)`: Displays SnackBar with amber/orange background.
- Each method should use `messengerKey.currentState?.showSnackBar(...)`.

### Step 2: Integrate in main.dart
- Modify `lib/main.dart`
- Import `toast_service.dart`.
- In `MaterialApp`, set `scaffoldMessengerKey: ToastService.messengerKey`.

### Step 3: Export in services.dart
- Modify `lib/services/services.dart`
- Add `export 'toast_service.dart';`.

## Technical Details
- **Approach:** Sử dụng `GlobalKey<ScaffoldMessengerState>` để điều khiển SnackBar toàn cục. Đây là pattern chuẩn trong Flutter khi cần tương tác UI từ tầng logic (Provider/Service).
- **Patterns to follow:** Tham khảo cách init các service khác trong `lib/services/storage_service.dart`.
- **Edge cases:** Xử lý trường hợp `messengerKey.currentState` là null (ví dụ: app chưa build xong hoặc đã dispose). Sử dụng null-aware operator `?.`.
