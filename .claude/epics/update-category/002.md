# Task: Repository & Provider CRUD Extensions

## Context
App cần một lớp dữ liệu vững chắc để quản lý Categories. Task này triển khai các phương thức CRUD trong Repository (bao gồm logic transaction an toàn khi xóa) và đồng bộ chúng lên Provider.

## Description
Mở rộng `RecordRepository` với các hàm thêm, sửa, xóa danh mục và tính toán tổng tiền bằng SQL. Cập nhật `RecordProvider` để cache dữ liệu và cung cấp API cho UI.

## Acceptance Criteria
- [ ] **FR-1 / Atomic Deletion:** Khi xóa Category, toàn bộ record thuộc nó được chuyển về ID 1 trong cùng một transaction.
- [ ] **FR-1 / ID 1 Protection:** Logic Repository ngăn chặn việc xóa hoặc sửa Category có ID 1.
- [ ] **NFR-2 / Performance:** Sử dụng SQL `SUM()` và `GROUP BY` để lấy tổng tiền nhanh chóng.
- [ ] **Data Integrity:** Tất cả Records đều phải có Category ID hợp lệ sau khi xóa một Category bất kỳ.

## Implementation Steps

### Step 1: Update RecordRepository
- Modify `lib/repositories/record_repository.dart`
- Thêm `Future<int> createCategory(Category category)`
- Thêm `Future<int> updateCategory(Category category)`
- Thêm `Future<int> deleteCategory(int id)`:
  - Sử dụng `database.transaction(...)`.
  - Step 1: `UPDATE Record SET category_id = 1 WHERE category_id = ?`.
  - Step 2: `DELETE FROM Category WHERE category_id = ?`.
  - Ném lỗi nếu `id == 1`.
- Thêm `Future<int> getRecordCountByCategoryId(int id)`: `SELECT COUNT(*) FROM Record WHERE category_id = ?`.
- Thêm `Future<Map<int, double>> getCategoryTotals()`:
  - Query: `SELECT category_id, SUM(amount) as total FROM Record GROUP BY category_id`.

### Step 2: Update RecordProvider
- Modify `lib/providers/record_provider.dart`
- Thêm `Map<int, double> _categoryTotals = {}`.
- Implement `Future<void> addCategory(Category category)`: Gọi repo, load lại dữ liệu, notify listeners.
- Implement `Future<void> updateCategory(Category category)`: Gọi repo, load lại dữ liệu, notify listeners.
- Implement `Future<void> deleteCategory(int id)`: Gọi repo, load lại dữ liệu, notify listeners.
- Cập nhật `loadAll()` để gọi `getCategoryTotals()` và cập nhật `_categoryTotals`.
- Thêm getter `double getCategoryTotal(int id) => _categoryTotals[id] ?? 0.0;`.

## Interface Contract
### Receives from 001:
- `ToastService` có thể được gọi trong Provider để báo lỗi nếu thao tác DB thất bại.

### Produces for 011:
- Methods `addCategory`, `updateCategory`, `deleteCategory` trong `RecordProvider`.
- Getter `getCategoryTotal(id)` để hiển thị số tiền trong List.

## Technical Details
- **Architecture:** Sử dụng SQL `SUM()` trực tiếp trong DB (NFR-2) để tối ưu hiệu năng thay vì tính toán thủ công trên mảng Records trong Dart.
- **Patterns to follow:** Tham khảo logic `createRecord` trong `RecordRepository` để viết transaction.
- **Edge cases:** Không cho phép xóa ID 1. Nếu `updateCategory` thất bại (tên trùng - dù đã check UI), cần bắt lỗi SQLite và báo qua Toast.
