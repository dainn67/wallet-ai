# Task: Verification & Edge Case Handling (ID 1 Protection)

## Context
Task cuối cùng để hoàn thiện logic an toàn cho Categories. Chúng ta cần đảm bảo Category "Uncategorized" (ID 1) không thể bị người dùng can thiệp xóa hay sửa.

## Description
Thêm các lớp bảo vệ tại cả UI và Repository cho Category ID 1. Viết Unit test để kiểm chứng luồng xóa danh mục và chuyển đổi giao dịch tự động.

## Acceptance Criteria
- [ ] **US-2 / ID 1 Immutable:** Không có nút Sửa/Xóa cho Category ID 1 trên UI.
- [ ] **US-2 / Repo Guard:** `RecordRepository.updateCategory` và `deleteCategory` trả về lỗi nếu `id == 1`.
- [ ] **US-2 / Correct Cleanup:** Toàn bộ Records thuộc danh mục bị xóa thực sự đã chuyển về `category_id = 1` trong DB.

## Implementation Steps

### Step 1: UI Protection
- Modify `lib/screens/home/tabs/categories_tab.dart`
- Ẩn nút Sửa/Xóa trong `ListTile` nếu `category.categoryId == 1`.
- Có thể thêm text "(Mặc định)" bên cạnh tên Uncategorized.

### Step 2: Repository Guard (Safety Net)
- Modify `lib/repositories/record_repository.dart`
- Step 1: Ném lỗi `ArgumentError` trong `deleteCategory(int id)` nếu `id == 1`.
- Step 2: Tương tự cho `updateCategory`.

### Step 3: Write Unit Test for Deletion
- Modify `test/repositories/record_repository_test.dart`
- Tạo test case:
  1. Thêm danh mục X.
  2. Tạo 5 records gán vào danh mục X.
  3. Xóa danh mục X.
  4. Verify: Danh mục X không còn trong DB.
  5. Verify: 5 records đó hiện tại có `category_id == 1`.

## Technical Details
- **Architecture:** "Security-in-depth" bằng cách chặn ở cả hai tầng (UI và Data).
- **Patterns to follow:** Sử dụng `assert` hoặc `throw` trong Repository cho các ràng buộc hệ thống.

## Tests to Write
### Unit Tests
- `test/repositories/record_repository_test.dart`
  - Test: `deleteCategory` cleanup logic works as expected.
  - Test: Attempting to delete ID 1 throws exception.
