# Task: Category Management UI (Add/Edit/Delete Dialogs)

## Context
Đây là phần giao diện chính giúp người dùng thao tác CRUD với danh mục. Task này xây dựng các Dialog để nhập tên mới, sửa tên và xác nhận xóa an toàn.

## Description
Hoàn thiện `CategoriesTab` bằng cách thêm các Dialog tương tác. Triển khai logic kiểm tra trùng tên ngay khi nhập liệu và hộp thoại xác nhận xóa có hiển thị số lượng giao dịch bị ảnh hưởng.

## Acceptance Criteria
- [ ] **FR-3 / Live Duplicate Check:** Hiện lỗi "Tên danh mục đã tồn tại" ngay khi người dùng nhập trùng.
- [ ] **US-2 / Safe Delete Dialog:** Dialog xóa hiển thị số lượng record sẽ bị chuyển về Uncategorized.
- [ ] **US-3 / List Totals:** Mỗi item trong danh sách hiển thị tổng số tiền (Amount) từ `RecordProvider.getCategoryTotal(id)`.
- [ ] **UI Polish:** List item có các nút sửa/xóa (trừ ID 1).

## Implementation Steps

### Step 1: Create CategoryFormDialog
- Modify `lib/components/components.dart` hoặc tạo mới `lib/components/dialogs/category_form_dialog.dart`.
- Dialog có một `TextFormField` cho tên danh mục.
- Logic validation: Check if `name` exists in `RecordProvider.categories` (case-insensitive).
- Thêm các nút "Hủy" và "Lưu".

### Step 2: Implement Edit/Delete Logic in CategoriesTab
- Modify `lib/screens/home/tabs/categories_tab.dart`
- Dùng `ListTile` để hiển thị mỗi Category.
- Thêm `trailing` widget với `IconButton(Icons.edit)` và `IconButton(Icons.delete, color: Colors.red)`.
- **Logic Xóa:**
  - Gọi `RecordRepository.getRecordCountByCategoryId(id)` để lấy số lượng records.
  - Hiển thị `ConfirmationDialog` với nội dung được custom: "Xóa danh mục này sẽ chuyển [số lượng] giao dịch về Uncategorized. Tiếp tục?".
  - Nếu xác nhận, gọi `recordProvider.deleteCategory(id)`.

### Step 3: Integrate Totals
- Trong `CategoriesTab`, hiển thị tổng tiền cho mỗi item: `Text(CurrencyHelper.format(recordProvider.getCategoryTotal(id)))`.

## Interface Contract
### Receives from 002:
- `RecordProvider.addCategory`, `updateCategory`, `deleteCategory`.
- `RecordProvider.getCategoryTotal(id)`.
- `RecordRepository.getRecordCountByCategoryId(id)`.

### Produces for 020:
- Giao diện Category Management đầy đủ.

## Technical Details
- **Architecture:** Dialogs nên là reusable components trong `lib/components/`.
- **Patterns to follow:** Tham khảo `ConfirmationDialog` hiện có trong codebase.
- **Edge cases:** Ngăn chặn hiển thị nút Xóa/Sửa cho Category có ID 1 (Uncategorized).
