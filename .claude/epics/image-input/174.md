# Task 020: Chat input attachment UI

## Context

Wally's chat input is currently a plain text-only pill-shaped `TextField`. This task adds an image attachment icon, a camera/gallery bottom sheet, a thumbnail preview strip showing chosen images, and wires everything into the updated `ChatProvider.sendMessage` (provided by T012) and `ImagePickerService` / `ImageProcessingService` (provided by T011 and T010). The goal UX is identical to how ChatGPT, Gemini, and Claude websites handle image input: pick, preview with the ability to remove, then send text + images together.

Architecture Decisions that apply:
- **AD-1:** Compress on pick (not on send) — thumbnails render from the already-compressed bytes.
- **AD-3:** No `permission_handler` — lazy prompts happen automatically via `image_picker`.
- **AD-5 / AD-6:** Oversize rejection and all-images-fail-send behavior are enforced here in the UI, using `OversizeImageException` from `ImageProcessingService`.

## Description

Modify `lib/screens/home/tabs/chat_tab.dart` to:
1. Add a photo icon (`Icons.add_photo_alternate_outlined`) at the trailing edge of the existing pill `TextField`.
2. On tap: open a `showModalBottomSheet` with "Take photo" and "Choose from library" rows.
3. On pick: compress each image via `ImageProcessingService.processPickedImage` (in parallel), append valid bytes to local state, show inline error banners for rejected images.
4. Render a `ImagePreviewStrip` widget (new file) above the input when images are pending.
5. On send: pass `imageBytes` to `ChatProvider.sendMessage`, then clear the list.
6. Update the send-enabled guard: allow send when text OR images present.

Create `lib/components/image_preview_strip.dart` as an extracted widget for the thumbnail row (keeps `chat_tab.dart` readable).

### Step 4: Implement `_pickFromCamera()` and `_pickFromGallery()`

  ```dart
  Future<void> _pickFromCamera() async {
    final file = await ImagePickerService().pickFromCamera();
    if (file == null || !mounted) return;
    await _processAndAdd([file]);
  }

  Future<void> _pickFromGallery() async {
    final remaining = 5 - _pendingImages.length;
    if (remaining <= 0) return;
    final files = await ImagePickerService().pickFromGallery(maxCount: remaining);
    if (!mounted || files.isEmpty) return;
    await _processAndAdd(files);
  }
  ```

### Step 5: Implement `_processAndAdd(List<XFile> files)`

  ```dart
  Future<void> _processAndAdd(List<XFile> files) async {
    final results = await Future.wait(
      files.map((f) async {
        try {
          final processed = await ImageProcessingService().processPickedImage(f);
          return processed.bytes;
        } on OversizeImageException {
          return null; // rejected — show error below
        } catch (_) {
          return null;
        }
      }),
    );

    final valid = results.whereType<Uint8List>().toList();
    final rejectedCount = results.length - valid.length;

    setState(() {
      _pendingImages.addAll(valid);
      if (rejectedCount > 0) {
        _attachErrorMessage = 'Image too large — $rejectedCount image(s) skipped.';
      }
    });

    // Auto-dismiss error after 3 seconds
    if (rejectedCount > 0) {
      Future.delayed(const Duration(seconds: 3), () {
        if (mounted) setState(() => _attachErrorMessage = null);
      });
    }
  }
  ```

### Step 6: Update `_handleSend()` to include images

  ```dart
  Future<void> _handleSend() async {
    final text = _controller.text.trim();
    final images = List<Uint8List>.from(_pendingImages);
    if (text.isEmpty && images.isEmpty) return; // AD-6: both empty → no-op

    _controller.clear();
    setState(() => _pendingImages.clear());

    try {
      FocusManager.instance.primaryFocus?.unfocus();
      await context.read<ChatProvider>().sendMessage(text, imageBytes: images.isEmpty ? null : images);
      WidgetsBinding.instance.addPostFrameCallback((_) => _scrollToBottom());
    } catch (e) {
      if (mounted) {
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(content: Text('Error: $e'), backgroundColor: Theme.of(context).colorScheme.error, behavior: SnackBarBehavior.floating),
        );
      }
    }
  }
  ```

## Technical Details

- **Font:** All text in `ImagePreviewStrip` uses the default (Poppins) — no explicit `fontFamily` needed as it's set globally in `ThemeData`.
- **`_controller` listener for send-enable:** Add in `initState`: `_controller.addListener(_onTextChanged);` and `void _onTextChanged() => setState(() {});`. Remove in `dispose`.
- **Streaming lock:** During streaming (`isStreaming == true`), the attachment icon is disabled (matches existing send-button behavior). Prevents picking images during an active stream.
- **Bottom sheet style:** Use `showModalBottomSheet` with `shape: RoundedRectangleBorder(borderRadius: BorderRadius.vertical(top: Radius.circular(16)))` for consistency with existing popup styling in the app.
