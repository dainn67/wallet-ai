# Task 011: ImagePickerService wrapper

## Context

Wally's chat input needs to invoke the device camera or photo gallery to let users attach images before sending a message. This task wraps the `image_picker` plugin in a testable singleton so the UI layer (T020) can be unit-tested with mocks instead of hitting native plugin code. The wrapper also enforces the 5-image cap at the API level. Permissions are lazy — `image_picker` requests them on first invocation; this service adds no additional permission logic (Architecture Decision AD-3 in the epic).

## Description

Create `lib/services/image_picker_service.dart` as a thin singleton wrapper around `ImagePicker` from the `image_picker` package. Expose two methods: `pickFromCamera()` returns a single nullable `XFile`; `pickFromGallery({required int maxCount})` returns a list of `XFile`, capped to `maxCount`. Follow the `setMockInstance` singleton pattern used by `ChatApiService` and `ImageProcessingService`.

## Acceptance Criteria

- [ ] **FR-2 / First-time camera tap:** Calling `pickFromCamera()` without prior permission triggers the native OS camera permission prompt (confirmed by `image_picker`'s built-in behavior — no additional code needed; verify manually during T090 QA).
- [ ] **FR-2 / Denied permission — graceful return:** When the user denies permission (or cancels the picker), `pickFromCamera()` and `pickFromGallery()` return `null` / empty list respectively — no exception thrown.
- [ ] **FR-3 / Multi-select cap:** `pickFromGallery(maxCount: 5)` returns at most 5 `XFile` entries even if the underlying picker returns more.
- [ ] **FR-3 / Count of 1:** `pickFromGallery(maxCount: 1)` returns at most 1 image.
- [ ] **Testability:** Unit tests can inject a mock `ImagePickerService` via `setMockInstance` and verify call behavior without hitting native code.

## Implementation Steps

### Step 1: Create the service file

- Create `lib/services/image_picker_service.dart`.
- Import `package:image_picker/image_picker.dart`.
- Implement singleton pattern (mirror `ChatApiService`):
  ```dart
  class ImagePickerService {
    static final _instance = ImagePickerService._internal();
    static ImagePickerService? _mockInstance;
    factory ImagePickerService() => _mockInstance ?? _instance;
    ImagePickerService._internal();
    static void setMockInstance(ImagePickerService? m) => _mockInstance = m;

    final _picker = ImagePicker();
    ...
  }
  ```

### Step 2: Implement `pickFromCamera()`

  ```dart
  Future<XFile?> pickFromCamera() async {
    return _picker.pickImage(source: ImageSource.camera);
  }
  ```
  - Returns `null` on cancel or permission denial (ImagePicker's native behavior).

### Step 3: Implement `pickFromGallery({required int maxCount})`

  ```dart
  Future<List<XFile>> pickFromGallery({required int maxCount}) async {
    final results = await _picker.pickMultiImage();
    if (results.isEmpty) return [];
    return results.take(maxCount).toList();
  }
  ```
  - `pickMultiImage()` is the `image_picker` API for multi-select gallery. Returns empty list on cancel.
  - `.take(maxCount)` enforces the caller-specified cap. The chat input widget passes `5 - currentCount` as `maxCount` so the cumulative total never exceeds 5.

### Step 4: Export from services barrel

- Open `lib/services/services.dart`.
- Add: `export 'image_picker_service.dart';`

### Step 5: Write unit tests

- Create `test/services/image_picker_service_test.dart`.
- Mock `ImagePickerService` using `mocktail`; verify `pickFromGallery(maxCount: 2)` with a mock that returns 5 items → result has length 2.
- Verify `pickFromCamera()` returns `null` when mock returns null.

## Technical Details

- **`image_picker` API:** `pickMultiImage()` returns `List<XFile>` (not nullable since v1.0). Returns empty list on cancel — no null check needed.
- **Permission prompts:** `image_picker` triggers the native prompt automatically on first use of `pickImage(source: ImageSource.camera)` and `pickMultiImage()`. No manual `requestPermission()` call is needed — this is the core of AD-3.
- **iOS gallery permission UX:** On iOS, `pickMultiImage` triggers `NSPhotoLibraryUsageDescription`. The permission string added in T001 will appear in this prompt.
- **maxCount cap note:** The cap is enforced client-side (`.take(maxCount)`), not at the OS picker level. On iOS 14+, the photo picker has a built-in selection limit; on Android and older iOS, the plugin returns all selected. The `.take()` ensures correctness regardless.
