# Task 021: Outgoing bubble image rendering and fullscreen viewer

## Context

After T012, user `ChatMessage` objects can carry `imageBytes: List<Uint8List>?`. The chat bubble widget (`lib/components/chat_bubble.dart`) currently only renders text — it needs to show image thumbnails for user messages that include images, and allow tapping a thumbnail to view it fullscreen. This task does not touch server communication or provider logic — it is purely UI rendering on top of data already wired up by T012.

Architecture Decision AD-4 (transient `imageBytes` on `ChatMessage`) means the bytes are available in memory during the session and rendered via `Image.memory` with no disk I/O.

## Description

Modify `lib/components/chat_bubble.dart` to render a horizontal row of image thumbnails above the existing text container for user-role messages with non-null `imageBytes`. Tapping a thumbnail opens `ImageViewer` — a new fullscreen widget wrapping `InteractiveViewer` + `Image.memory`. Create `lib/components/image_viewer.dart` and export it from the components barrel.

## Acceptance Criteria

- [ ] **FR-6 / Happy path — bubble layout:** A user message with 2 images and caption "breakfast" renders both thumbnails above (or surrounding) the text container in the outgoing bubble.
- [ ] **FR-6 / Caption-only message (no regression):** A user message with no `imageBytes` renders exactly as before — text-only bubble with no thumbnail row.
- [ ] **FR-6 / Tap to view — fullscreen opens:** Tapping any thumbnail opens a full-screen black-background viewer displaying the image via `Image.memory`; back button / swipe dismisses it.
- [ ] **FR-6 / Images-only message (no text):** A user message with images and empty caption renders thumbnails with no text content below.
- [ ] **No regression on assistant bubbles:** Assistant-role messages still render as text + optional record widgets + suggestion banners — no change.
- [ ] **No regression on analysis indicator:** `message.isAnalyzing` spinner still renders correctly for assistant messages.

## Implementation Steps

### Step 1: Add thumbnail row to `ChatBubble` for user-role messages

- Open `lib/components/chat_bubble.dart`.
- Add import: `import 'dart:typed_data';` and `import 'image_viewer.dart';`
- In the `build` method, inside the `Flexible → Column` that wraps the bubble, add a thumbnail row **above** the existing text `Container` when `isUser && message.imageBytes != null && message.imageBytes!.isNotEmpty`:

  ```dart
  if (isUser && message.imageBytes != null && message.imageBytes!.isNotEmpty) ...[
    _buildThumbnailRow(context, message.imageBytes!),
    const SizedBox(height: 8),
  ],
  ```

### Step 2: Implement `_buildThumbnailRow`

  ```dart
  Widget _buildThumbnailRow(BuildContext context, List<Uint8List> images) {
    return Wrap(
      spacing: 6,
      runSpacing: 6,
      alignment: WrapAlignment.end,
      children: images.asMap().entries.map((e) {
        return GestureDetector(
          onTap: () => Navigator.of(context).push(
            MaterialPageRoute(builder: (_) => ImageViewer(imageBytes: e.value)),
          ),
          child: ClipRRect(
            borderRadius: BorderRadius.circular(8),
            child: Image.memory(e.value, width: 96, height: 96, fit: BoxFit.cover),
          ),
        );
      }).toList(),
    );
  }
  ```

  - `Wrap` handles layout automatically — if 3 images fit in a row they line up; if 4+ they wrap to a second row.
  - `WrapAlignment.end` aligns with the right-side user bubble.

### Step 3: Handle caption-only render (no regression)

- The existing text `Container` should still render when `message.content.trim().isNotEmpty`.
- If content is empty and images are present (images-only message), skip the text container:
  ```dart
  if (message.content.trim().isNotEmpty)
    Container(
      padding: ...,
      decoration: ...,
      child: message.isAnalyzing ? /* spinner row */ : Text(message.content.trim(), ...),
    ),
  ```
- This replaces the unconditional `Container(...)` with a conditional one. All existing styling inside remains unchanged.

### Step 4: Create `lib/components/image_viewer.dart`

  ```dart
  import 'dart:typed_data';
  import 'package:flutter/material.dart';

  class ImageViewer extends StatelessWidget {
    final Uint8List imageBytes;
    const ImageViewer({super.key, required this.imageBytes});

    @override
    Widget build(BuildContext context) {
      return Scaffold(
        backgroundColor: Colors.black,
        appBar: AppBar(
          backgroundColor: Colors.black,
          iconTheme: const IconThemeData(color: Colors.white),
          elevation: 0,
        ),
        body: Center(
          child: InteractiveViewer(
            panEnabled: true,
            minScale: 0.5,
            maxScale: 4.0,
            child: Image.memory(imageBytes, fit: BoxFit.contain),
          ),
        ),
      );
    }
  }
  ```

  - `InteractiveViewer` provides pinch-to-zoom and pan for free — no additional dependencies.
  - `AppBar` with default back arrow handles navigation dismissal.

### Step 5: Export from components barrel

- Open `lib/components/components.dart`.
- Add: `export 'image_viewer.dart';`

## Interface Contract

### Receives from T012:
- File: `lib/models/chat_message.dart`

## Technical Details

- **`Image.memory` usage:** Accepts `Uint8List` directly. Uses cached decode by Flutter's image framework — fast for thumbnails.
- **`Wrap` vs `ListView`:** `Wrap` is preferred over a horizontal `ListView` for the thumbnail grid because it handles overflow to multiple rows gracefully when many images are present, matches the natural flow of the bubble, and requires no height constraint.
- **No hero animation:** AD-1 (compress on pick) means bytes are in memory from pick time — no URL, no hero tag needed. Simple `MaterialPageRoute` push is sufficient for v1.
- **Caption-only text rendering:** The conditional `if (message.content.trim().isNotEmpty)` wrapping the text `Container` is the minimal change needed. Existing styling, `isAnalyzing` check, and record widgets are all inside or below this container and are unaffected.
- **Thumbnail size:** 96×96 matches the `ImagePreviewStrip` preview in the input (T020) — consistent visual language.
