# Task: Global Theme Update & UI Refactoring

## Context
After registering the local Poppins fonts, the app must be updated to use them as the default and remove the dependency on `google_fonts`.

## Description
Set `fontFamily: 'Poppins'` in the global `ThemeData` within `lib/main.dart`. Replace all instances of `GoogleFonts.poppins(...)` with standard `TextStyle(...)` or similar across the codebase. Finally, remove `google_fonts` from `pubspec.yaml` and delete all imports of the library.

## Acceptance Criteria
- [x] **FR-2 / Happy path:** App renders with Poppins font without `google_fonts` active.
- [x] **FR-3 / Clean dependencies:** `google_fonts` is removed from `pubspec.yaml`.
- [x] **FR-3 / Compilation:** App compiles and runs without errors.
- [x] **NFR-1 / Performance:** App starts without network fetching for fonts.

## Implementation Steps

### Step 1: Update Global Theme
- Modify `lib/main.dart`:
  - Set `fontFamily: 'Poppins'` inside `ThemeData`.
  - Replace `textTheme: GoogleFonts.poppinsTextTheme(...)` with a standard `TextTheme` or simply remove the line to let the global `fontFamily` handle it.
  - Update `AppBarTheme.titleTextStyle` to use standard `TextStyle`.

### Step 2: Refactor UI Components
- Search for `GoogleFonts.poppins` in all `lib/**/*.dart` files.
- Replace them with `TextStyle` or the appropriate theme-based text style.
  - Example: `style: GoogleFonts.poppins(fontSize: 14)` becomes `style: TextStyle(fontSize: 14)`.
- Remove `import 'package:google_fonts/google_fonts.dart';` from all files.

### Step 3: Remove Dependency
- Open `pubspec.yaml`.
- Remove `google_fonts: ^6.2.1` from `dependencies`.
- Run `fvm flutter pub get`.

## Technical Details
- **Approach:** Global theme override and manual widget refactoring.
- **Files to create/modify:** `lib/main.dart`, and multiple component/screen files.
- **Patterns to follow:** Use `Theme.of(context).textTheme` where appropriate instead of hardcoded `TextStyle`.
- **Edge cases:** Widgets that were using `GoogleFonts` properties that might not perfectly map to `TextStyle` (though Poppins is consistent).
