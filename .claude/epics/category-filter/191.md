# Task: Integration verification & cleanup

## Context

Final quality gate before the category-filter epic is marked complete. Verifies all four tasks integrate correctly end-to-end, no regressions in existing tabs, all PRD requirements are met, and the diff satisfies the code-footprint constraint (NFR-2: ≤1 new file).

## Acceptance Criteria

- [ ] **FR-1 / End-to-end:** Tapping a parent category row on the Categories tab opens `CategoryRecordsBottomSheet` with the union of parent + sub records for the current month.
- [ ] **FR-1 / Sub-tap:** Tapping a sub-category row opens the popup scoped to that sub only.
- [ ] **FR-2 / Grouped view:** Parent popup shows one bordered section per sub-category + one for parent-direct records; sub popup is a flat list.
- [ ] **FR-3 / Edit round-trip:** Editing a record from the popup saves correctly; popup refreshes without manual close/reopen.
- [ ] **FR-4 / Sort order:** Records tab and popup both sort newest-first by `occurredAt`; editing an old record does not move it to the top.
- [ ] **NTH-1 / Empty state:** Category with no records in the current month shows the empty-state message in the popup.
- [ ] **NFR-2 / One new file:** `git diff main --name-only` shows exactly one new file (`category_records_bottom_sheet.dart`).
- [ ] **Regression — Records tab:** Records tab still loads, displays, and allows edit/delete without errors.
- [ ] **Regression — Categories tab:** Add category, edit category, add sub-category, delete category all still function correctly.
- [ ] **Regression — Chat tab:** Record creation via AI still works; suggested category banners still appear.
- [ ] **All tasks complete:** Tasks 001, 010, 020 are all `status: done`.

### Step 4: Manual golden path (device or simulator)
- Open Categories tab, pick a month with known records.
- Tap a parent category with sub-categories → verify grouped popup with borders.
- Tap a sub-category → verify flat popup scoped to sub.
- In popup: tap edit on a record → edit description → save → verify popup refreshes.
- Dismiss popup → tap chevron on parent → verify expand/collapse works.
- Tap pencil icon on sub row → verify category edit dialog (not records popup).
- Open Records tab → scroll to an old record → edit description → verify position unchanged.

### Step 5: Edge case checks
- Tap "Uncategorized" (categoryId=1) row → popup opens (may show 0 or more records); no crash.
- Tap a parent with no sub-categories → popup opens as flat list with no extra borders.
- Tap a category that has records from a different month (outside `selectedDateRange`) → those records do NOT appear in popup.
- Scroll popup list to 50+ records (if available) → no jank.
