# Task: Rewire Categories tab — row tap opens popup, chevron expands

## Context

The Categories tab (`lib/screens/home/tabs/categories_tab.dart`) currently has two conflicting tap behaviors: parent rows are wrapped in `ExpansionTile` which handles tap to expand/collapse, and sub rows call `_showEditDialog` on tap. The new behavior replaces both: tapping any category or sub-category row opens the `CategoryRecordsBottomSheet` (from task 010), while expansion/collapse moves exclusively to a dedicated trailing icon button driven by `ExpansionTileController`. Sub-category edit moves to the `onEdit` pencil icon that `CategoryWidget` already supports. The `CategoriesTab` class is currently a `StatelessWidget` and must be converted to `StatefulWidget` to hold the `ExpansionTileController` map.

## Description

Convert `CategoriesTab` to `StatefulWidget`. Add a `Map<int, ExpansionTileController> _controllers` field initialized per parent category. Change parent `CategoryWidget.onTap` from `null` to `_openCategoryPopup(category, isParent: true)` — `CategoryWidget`'s inner `InkWell` absorbs the tap before `ExpansionTile`'s header handler, so the tile does NOT toggle on row tap. Move expand/collapse to a trailing `IconButton` calling `controller.toggle()`. For sub rows, change `onTap` to `_openCategoryPopup(sub, isParent: false)` and add `onEdit: () => _showEditDialog(context, sub)` so the edit pencil icon appears.

## Acceptance Criteria

- [ ] **FR-1 / Sub tap → popup:** Tapping a sub-category row opens `CategoryRecordsBottomSheet` scoped to that sub only.
- [ ] **FR-1 / Chevron still works:** Tapping the trailing chevron/arrow icon toggles the `ExpansionTile` open/closed independently of the row tap.
- [ ] **FR-2 / Union popup:** Parent popup receives `categoryIds` = [parentId, sub1Id, sub2Id, ...] and `subCategories` = list of subs — verifiable by reading `_openCategoryPopup` call.
- [ ] **FR-3 / Sub edit preserved:** The pencil icon on sub-category rows still opens the edit-category dialog (not the records popup).
- [ ] **Regression / Parent edit preserved:** The pencil icon on parent rows (non-Uncategorized) still opens the edit-category dialog.
- [ ] **Regression / Add sub still works:** The "+ Add sub-category" button at the bottom of each expanded tile still functions.
- [ ] **Regression / Uncategorized row:** Tapping the Uncategorized row opens its records popup (not blocked); it has no edit pencil since `onEdit` is `null` for `categoryId == 1`.

## Implementation Steps

### Step 1: Convert CategoriesTab to StatefulWidget
- Open `lib/screens/home/tabs/categories_tab.dart`
- Change `class CategoriesTab extends StatelessWidget` to:
  ```dart
  class CategoriesTab extends StatefulWidget {
    const CategoriesTab({super.key});
    @override
    State<CategoriesTab> createState() => _CategoriesTabState();
  }
  class _CategoriesTabState extends State<CategoriesTab> {
    final Map<int, ExpansionTileController> _controllers = {};
    // move all existing methods and build() here
  }
  ```
- Move `_showAddDialog`, `_showEditDialog`, and `build()` into `_CategoriesTabState`.

### Step 2: Initialize controllers in build
- At the top of `build()`, after computing `parentCategories`, populate controllers:
  ```dart
  for (final cat in parentCategories) {
    _controllers.putIfAbsent(cat.categoryId!, () => ExpansionTileController());
  }
  ```
  Using `putIfAbsent` preserves existing controllers (and their expanded state) across rebuilds.

### Step 3: Add `_openCategoryPopup` helper
- Add private method to `_CategoriesTabState`:
  ```dart
  void _openCategoryPopup(BuildContext context, Category category, {required bool isParent}) {
    final provider = context.read<RecordProvider>();
    final subCats = isParent ? provider.getSubCategories(category.categoryId!) : <Category>[];
    final ids = [category.categoryId!, ...subCats.map((s) => s.categoryId!)];
    showModalBottomSheet(
      context: context,
      isScrollControlled: true,
      backgroundColor: Colors.transparent,
      builder: (_) => CategoryRecordsBottomSheet(
        category: category,
        categoryIds: ids,
        subCategories: subCats,
      ),
    );
  }
  ```
- Import `category_records_bottom_sheet.dart`.

### Step 4: Update parent CategoryWidget in ExpansionTile
- Find the `CategoryWidget` inside `ExpansionTile(...title: CategoryWidget(...))` block.
- Change `onTap: null` → `onTap: () => _openCategoryPopup(context, category, isParent: true)`
- Add `controller: _controllers[category.categoryId!]` to `ExpansionTile`.
- Replace `ExpansionTile`'s default trailing (the built-in chevron) by adding:
  ```dart
  trailing: IconButton(
    icon: const Icon(Icons.keyboard_arrow_down_rounded, color: Color(0xFF94A3B8)),
    onPressed: () => _controllers[category.categoryId!]?.toggle(),
    padding: EdgeInsets.zero,
    constraints: const BoxConstraints(),
  ),
  ```
  Note: setting a custom `trailing` overrides the default expand arrow. The icon does not rotate on expand — if you want rotation, wrap in `AnimatedRotation` controlled by a local bool or listen to `ExpansionTile.onExpansionChanged`.
- Keep `onEdit: category.categoryId == 1 ? null : () => _showEditDialog(context, category)` unchanged.

### Step 5: Update sub-category CategoryWidget rows
- Find the `subCategories.map((sub) => CategoryWidget(...))` block.
- Change `onTap: () => _showEditDialog(context, sub)` → `onTap: () => _openCategoryPopup(context, sub, isParent: false)`
- Add `onEdit: () => _showEditDialog(context, sub)` — `CategoryWidget` renders the pencil icon when `onEdit != null`, so edit dialog is now accessible via the pencil.

### Step 6: Verify no regressions
- Confirm `showAddSubCategoryDialog` call (the "+ Add sub-category" button) is unchanged.
- Confirm `_showAddDialog` (the top-right add button) is unchanged.

## Technical Details

- **InkWell absorption (AD-2 from epic):** When `CategoryWidget.onTap` is non-null, its `InkWell` consumes the tap gesture. `ExpansionTile`'s built-in header tap handler uses a `GestureDetector` higher in the tree, but the `InkWell` in the title widget absorbs the pointer first. This is tested Flutter behavior — the tile does NOT expand when the title's InkWell handles the tap.
- **ExpansionTileController:** Requires `controller:` to be passed to `ExpansionTile`. The controller's `toggle()`, `expand()`, and `collapse()` methods are called imperatively from the trailing button.
- **`putIfAbsent` for controllers:** Controllers are created lazily and preserved across list rebuilds (e.g., when a category is renamed). Avoids losing expansion state on `notifyListeners()`.
- **Existing `_showEditDialog`** opens `CategoryFormDialog` (not `EditRecordPopup`) — it edits the category's name/type, not a transaction. Sub rows now access this via pencil icon instead of row tap.
- **`StatelessWidget` → `StatefulWidget` conversion** is mechanical: wrap existing build content in a State class. The `Consumer<RecordProvider>` pattern inside `build()` is unchanged.
