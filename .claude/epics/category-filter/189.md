# Task: Build CategoryRecordsBottomSheet widget

## Context

The Categories tab needs a drill-down popup to show individual records behind each category's monthly total. This task creates the one new widget file (`CategoryRecordsBottomSheet`) that the tab will open on row tap. It reuses `RecordWidget` (existing, `lib/components/record_widget.dart`) for record rows and `EditRecordPopup` (existing, `lib/components/popups/edit_record_popup.dart`) for the edit flow — no new UI patterns introduced. PRD NFR-2 hard-caps this at one new widget file.

## Description

Create `lib/components/popups/category_records_bottom_sheet.dart` as a `StatelessWidget`. It accepts `category` (the tapped item), `categoryIds` (ids to include — caller builds the union), and `subCategories` (empty for sub-tap, populated for parent-tap to build group headers). Inside a `Consumer<RecordProvider>`, it calls `provider.getRecordsForCategory(categoryIds, provider.selectedDateRange)` so the list auto-refreshes after edits. Renders a `DraggableScrollableSheet` with grouped, bordered sections for parent-tap and a flat list for sub-tap. Each record row is a `RecordWidget` with `onEdit` wired to `EditRecordPopup`.

## Acceptance Criteria

- [ ] **FR-2 / Grouped parent view:** When opened with `subCategories: [Groceries, Dining]`, the popup shows 3 bordered sections: parent-direct records, Groceries records, Dining records — in that order.
- [ ] **FR-2 / Flat sub view:** When opened with `subCategories: []`, the popup shows a flat list of records with no group borders.
- [ ] **FR-3 / Edit round-trip:** Tapping the edit button on a record row opens `EditRecordPopup`; after saving, the popup list updates to show the new values without manual refresh.
- [ ] **NTH-1 / Empty state:** When `getRecordsForCategory` returns an empty list, the popup body shows a centered message (e.g., "No records in this category for [month].").
- [ ] **NFR-1 / Latency:** Popup fully renders within 200ms for a scope containing 100 records (plain `ListView`, no pagination).
- [ ] **NFR-2 / Single file:** Diff shows exactly one new file: `lib/components/popups/category_records_bottom_sheet.dart`.

## Implementation Steps

### Step 1: Create file and widget skeleton
- Create `lib/components/popups/category_records_bottom_sheet.dart`
- Define `CategoryRecordsBottomSheet extends StatelessWidget` with constructor:
  ```dart
  const CategoryRecordsBottomSheet({
    super.key,
    required this.category,
    required this.categoryIds,
    required this.subCategories,
  });
  final Category category;
  final List<int> categoryIds;
  final List<Category> subCategories;
  ```
- Required imports: `record_provider.dart`, `record_widget.dart`, `edit_record_popup.dart`, `category.dart`, `record.dart`

### Step 2: Build outer scaffold using DraggableScrollableSheet
- `build()` wraps in `Consumer<RecordProvider>`:
  ```dart
  return Consumer<RecordProvider>(
    builder: (context, provider, _) {
      final records = provider.getRecordsForCategory(categoryIds, provider.selectedDateRange);
      return DraggableScrollableSheet(
        initialChildSize: 0.6,
        maxChildSize: 0.95,
        minChildSize: 0.3,
        expand: false,
        builder: (_, scrollController) => Column(
          children: [
            _buildHeader(context, provider),
            Expanded(child: _buildBody(context, records, scrollController)),
          ],
        ),
      );
    },
  );
  ```
- `_buildHeader`: a drag handle pill + `Text(category.name)` (bold) + current month label derived from `provider.selectedDateRange?.start` formatted as "MMM yyyy".

### Step 3: Build grouped body for parent-tap
- `_buildBody(context, records, scrollController)`:
  - If `records.isEmpty`: return `Center(child: Text('No records in this category for ${_monthLabel(provider.selectedDateRange)}.'))`
  - If `subCategories.isEmpty`: flat `ListView(controller: scrollController, children: records.map((r) => _buildRecordRow(context, r)).toList())`
  - If `subCategories.isNotEmpty`: `ListView(controller: scrollController)` with sections:
    1. Parent-direct section: `records.where((r) => r.categoryId == category.categoryId!)`
    2. Per sub: `records.where((r) => r.categoryId == sub.categoryId!)` for each sub in `subCategories`
    - Each section: `Container(margin: ..., decoration: BoxDecoration(border: Border.all(color: Color(0xFFE2E8F0)), borderRadius: BorderRadius.circular(12)), child: Column([headerText, ...rows]))`
    - Skip sections with 0 records (no empty bordered boxes)

### Step 4: Build each record row
- `_buildRecordRow(BuildContext context, Record record)`:
  ```dart
  RecordWidget(
    record: record,
    isEditable: true,
    onEdit: () => _showEditPopup(context, record),
  )
  ```
- `_showEditPopup`:
  ```dart
  void _showEditPopup(BuildContext context, Record record) {
    showModalBottomSheet(
      context: context,
      isScrollControlled: true,
      builder: (_) => EditRecordPopup(record: record),
    );
  }
  ```
  This mirrors `RecordsTab._showEditRecordPopup` exactly.

### Step 5: Month label helper
- Private `String _monthLabel(DateTimeRange? range)` — returns `'MMM yyyy'` from `range?.start` using `intl` or a simple manual format (check existing date formatting in the codebase first; follow whatever pattern RecordsTab uses).

## Interface Contract

### Receives from Task 001:
- Provider method: `provider.getRecordsForCategory(List<int> categoryIds, DateTimeRange? range)` → `List<Record>` sorted `occurredAt DESC`
- Provider field: `provider.selectedDateRange` → `DateTimeRange?` (the tab's current month)
- Invariants: returned list is never null; always sorted newest-first; records with `categoryId` not in `categoryIds` are never included.

### Produces for Task 020:
- Widget: `CategoryRecordsBottomSheet({required category, required categoryIds, required subCategories})`
- Usage: `showModalBottomSheet(isScrollControlled: true, builder: (_) => CategoryRecordsBottomSheet(...))`

## Technical Details

- **Reuse pattern:** `RecordWidget` already has `onEdit: VoidCallback?` (line 16 of `record_widget.dart`) and renders an edit `IconButton` at line 101. Pass `onEdit` to enable it — no modification to `RecordWidget` needed.
- **Auto-refresh:** Because the widget is a `Consumer<RecordProvider>`, any `notifyListeners()` from `RecordProvider` (triggered by `EditRecordPopup` saving) causes the bottom-sheet to rebuild and re-query `getRecordsForCategory` — fresh data without manual state.
- **Group ordering:** Parent-direct group always first, then subs in the order they appear in `subCategories` (which matches the tab's display order).
- **Border style:** Use `Border.all(color: Color(0xFFE2E8F0))` to match the existing card border style in `category_widget.dart` line 47.
- **Empty sections:** Skip rendering a group entirely if it has 0 records — avoids empty bordered boxes.
