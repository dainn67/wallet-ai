# Task: Refactor AI Parser

## Context
Simplify the parsing logic by using IDs provided by the AI.

## Description
Rewrite the `onDone` JSON parsing logic in `ChatProvider`. It should now expect `source_id` and `category_id` in the JSON response. Remove the legacy `getMoneySourceByName` logic.

## Implementation Steps
- Step 1: Locate the parsing loop in `ChatProvider.sendMessage.onDone`.
- Step 2: Update the logic to extract `source_id` and `category_id` from the AI's JSON item.
- Step 3: Use these IDs directly to create the `Record` object.
- Step 4: Implement graceful fallbacks: if an ID is missing or invalid, default to ID 1.
- Step 5: Delete the `getMoneySourceByName` and auto-creation logic for missing sources.

## Acceptance Criteria
- [ ] **FR-4 / Parser:** Records are created using IDs from the JSON.
- [ ] **FR-5 / Fallbacks:** No crashes if IDs are missing; uses default ID 1.
- [ ] **Cleanup:** All string-matching code is removed.

## Verification
- Unit test the parser with mock JSON containing `source_id` and `category_id`.
