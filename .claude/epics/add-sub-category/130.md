# Task: Database Schema & Migration

## Context
Update the SQLite database to version 7 to support sub-categories and migration logic.

## Interface Contract
- **Receives from T129**: Updated `Category` model with `parentId`.

## Acceptance Criteria
- [ ] DB version is bumped to 7.
- [ ] `onUpgrade` adds `parent_id` column to `Category` table (defaulting to -1).
- [ ] `_seedDatabase` is updated with a hierarchy: Parents (ID 1-8) and their respective sub-categories.
- [ ] `getAllRecords` JOIN logic updated to fetch "Parent - Sub" name format.

## Implementation Steps
1.  **Modify `lib/repositories/record_repository.dart`**:
    *   Update `_dbVersion` to 7.
    *   Add migration in `_onUpgrade`: `ALTER TABLE Category ADD COLUMN parent_id INTEGER DEFAULT -1;`.
    *   Update `_onCreate` schema definition for `Category`.
    *   Refactor `_seedDatabase` to include the hierarchical structure (Food, Transport, etc. as parents).
2.  **Refactor Queries**:
    *   In `getAllRecords`, update the `SELECT` query to JOIN with `Category` (let's call it `sub`) and again with `Category` (alias `parent`) using `sub.parent_id = parent.category_id`.
    *   Format the `category_name` as `COALESCE(parent.name || ' - ' || sub.name, sub.name)`.

## Verification
- Perform a clean app install and verify seeded data.
- Verify migration from v6 to v7 by manually triggering an upgrade in a test environment.
- Run `fvm flutter test test/repositories/record_repository_test.dart`.
