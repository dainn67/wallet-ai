# Task: Implement Popup Validation

## Context
Inputs from the user must be validated before attempting database operations. This task ensures the "Add Source" data is correctly formatted and complete.

## Description
Add validation logic to `AddSourcePopup.dart`. Ensure the "Source Name" is not empty and "Initial Amount" is a valid, positive number.

## Implementation Steps
1. Open `lib/components/popups/add_source_popup.dart`.
2. Wrap inputs in a `Form` or add custom validation logic to the "Save" button `onPressed` handler.
3. Check:
   - `name.trim().isNotEmpty`.
   - `double.tryParse(amountString) != null`.
4. If invalid, show an error message (using `errorText` in `InputDecoration` or a `ScaffoldMessenger` snackbar).
5. Only call `Navigator.pop(context, result)` if the data is valid.

## Acceptance Criteria
- [ ] **FR-2 / scenario:** Saving with an empty name or invalid amount fails and shows an error message.
- [ ] **FR-2 / scenario:** Valid inputs allow the popup to close and return a value.

## Tests to Write
- Unit tests for the validation logic (if extracted to a helper) or widget tests verifying error message presence for invalid inputs.

## Verification Checklist
- [ ] Attempt to save with empty fields and verify error appearance.
- [ ] Attempt to save with "abc" as amount and verify error appearance.
- [ ] Attempt to save with valid data and verify the popup closes.
