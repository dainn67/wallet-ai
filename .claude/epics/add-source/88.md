# Task: Database Integration and Record Generation

## Context
Adding a new source with an initial balance must be atomic and traceable. This task implements the database operations to create the `MoneySource` and its corresponding "Initial Balance" record.

## Description
Update `RecordRepository` (or equivalent service) to include a method for adding a source with an optional initial balance. If the amount is greater than zero, it should also create an 'income' record in the `record` table.

## Implementation Steps
1. Open `lib/repositories/record_repository.dart`.
2. Review existing `createMoneySource` method.
3. Modify `createMoneySource` (or add a wrapper like `addSourceWithBalance`) to accept `amount`.
4. Use `database.transaction` to:
   - Insert new `MoneySource` (get the `source_id`).
   - If `amount > 0`:
     - Create a `Record` object with `type: 'income'`, `description: 'Initial Balance'`, `moneySourceId: source_id`, `amount: amount`, `categoryId: 1` (Uncategorized or system ID).
     - Insert the `Record` into the `record` table.
5. Error handling: Catch any DB exceptions and rethrow to be handled by the UI.

## Acceptance Criteria
- [ ] **FR-3 / scenario:** Saving a new source with amount > 0 inserts a row in `MoneySource` AND a row in `record`.
- [ ] **FR-3 / scenario:** Saving a new source with amount 0 only inserts a row in `MoneySource`.
- [ ] **Data Integrity:** Both operations happen within a single database transaction.

## Tests to Write
- Unit test for `RecordRepository.createMoneySource` verifying that both entries are created when an amount is provided.

## Verification Checklist
- [ ] Add a new source via the app and verify the new source exists in the list.
- [ ] Manually check the `record` table (if possible) for the "Initial Balance" entry.
- [ ] Verify the total balance correctly includes the new source's amount.
