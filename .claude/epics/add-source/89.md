# Task: Refresh UI after Source Addition

## Context
After a new source is added to the database, the UI should immediately reflect the change by showing the new source in the `RecordsOverview` list.

## Description
Connect the `RecordsOverview` add button to the `AddSourcePopup` and ensure that saving the popup triggers a data refresh in the `RecordProvider`.

## Implementation Steps
1. Open `lib/components/records_overview.dart`.
2. Update the `IconButton`'s `onPressed` callback:
   - Call `showDialog(context: context, builder: (context) => AddSourcePopup())`.
   - Wait for the result (e.g., a `MoneySource` or a signal).
3. If a new source was added successfully:
   - Call `recordProvider.fetchData()` (or the equivalent refresh method in `lib/providers/record_provider.dart`).
4. Ensure `notifyListeners()` is called in the provider to trigger a rebuild of the `RecordsOverview` component.

## Acceptance Criteria
- [ ] **FR-3 / scenario:** After a source is successfully added via the popup, the `RecordsOverview` list updates automatically to include it.
- [ ] **UI Refresh:** The total balance, income, and sources list reflect the new data without a full screen restart.

## Tests to Write
- Integration test for adding a source and verifying that the `RecordProvider`'s state is updated correctly.

## Verification Checklist
- [ ] Open the app and go to the Records tab.
- [ ] Add a new source with a name like "Pocket Money".
- [ ] Verify that "Pocket Money" immediately appears in the horizontal source list.
