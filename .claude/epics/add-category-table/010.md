# Task: Update Provider with Caching

## Context
Implement performant category lookups in the application state.

## Description
Modify `RecordProvider` to:
1. Fetch all categories from the repository during initialization.
2. Store categories in a `List<Category>` and a `Map<int, String> _categoryCache` for O(1) name lookups.
3. Expose a `getCategoryName(int id)` helper method.

## Implementation Steps
- Step 1: Add category list and map to `RecordProvider`.
- Step 2: Update `loadAll()` or initialization logic to populate these.
- Step 3: Ensure the cache is refreshed if categories change (though out of scope for now).

## Acceptance Criteria
- [ ] **FR-4 / Caching:** Category names are available in the Provider without repeated DB calls.
- [ ] **FR-4 / Logic:** `RecordProvider` correctly initializes with category data.
