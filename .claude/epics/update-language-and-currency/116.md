# Task: Localize Core UI

## Context
All hardcoded strings in the UI must be replaced with dynamic lookups from the `LocaleProvider` to support switching.

## Description
Refactor core screens and tabs to use localized strings. Ensure the UI updates immediately when the language is changed in the drawer.

## Acceptance Criteria
- [ ] **FR-2 / UI:** Drawer labels, tab titles, record headers, and button texts are localized.
- [ ] **NFR-1 / Latency:** UI updates reactively when language changes without manual restart.

## Implementation Steps
1. Locate hardcoded strings in `home_screen.dart`, `chat_tab.dart`, `records_tab.dart`.
2. Replace strings with `context.watch<LocaleProvider>().translate('key')`.
3. Add a language selector (e.g., `Switch` or `Dropdown`) in the Drawer.
4. Verify all common popups (`EditSourcePopup`, `ConfirmationDialog`) also use the translation system.

## Tests to Write
- Widget test: `test/screens/home/home_localization_test.dart` to verify text changes when locale provider state is toggled.
