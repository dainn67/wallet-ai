# Task: Implement LocaleProvider and Persistence

## Context
We need a reactive way to manage the current language and currency and ensure they persist across app restarts.

## Description
Create `LocaleProvider` as a `ChangeNotifier`. It will hold the `currentLanguage` and `currentCurrency` and handle updates. Use `StorageService` (SharedPreferences) for persistence.

## Acceptance Criteria
- [ ] **FR-2 / State:** `LocaleProvider` notifies listeners when language or currency changes.
- [ ] **FR-5 / Persistence:** Language and currency are loaded from `SharedPreferences` on initialization and saved on change.
- [ ] **FR-2 / Helper:** `translate(String key)` method returns the correct string for the active language.

## Implementation Steps
1. Create `lib/providers/locale_provider.dart`.
2. Implement `LocaleProvider extends ChangeNotifier`.
3. Add properties: `AppLanguage _language`, `AppCurrency _currency`.
4. Implement `init()`: Load values from `StorageService`.
5. Implement `setLanguage(AppLanguage lang)` and `setCurrency(AppCurrency curr)`: Update state, notify listeners, and save to storage.
6. Implement `String translate(String key) => L10nConfig.translations[_language]![key] ?? key;`.
7. Register `LocaleProvider` in `lib/main.dart`'s `MultiProvider`.

## Tests to Write
- Unit test: `test/providers/locale_provider_test.dart`.
  - Verify switching language/currency notifies listeners.
  - Verify `translate` returns correct values for different locales.
  - Verify persistence via mocked `StorageService`.
