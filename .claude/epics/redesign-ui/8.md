# T8: Popups & dialogs (11 surfaces)

## Context
Every modal surface in the app needs to migrate to the new design language. The 11 surfaces share a common pattern: themed `Dialog`/`showModalBottomSheet` container, `FilledButton` primary action, `OutlinedButton` or `TextButton` secondary, themed `InputDecoration` text fields. This is a long-tail task — 11 surfaces at ~30 min each — so it is structured as 11 sub-checkboxes. Each popup commit is independent. All save/delete/dismiss callbacks are byte-equivalent (no behavior changes whatsoever). The `OnboardingDialog` visual migration is included here; its content/step logic changes are in T9.

## Description
Apply the redesign-ui design language to all 11 popup surfaces in `lib/components/popups/`. Each popup gets the same treatment:
1. Container: themed `Dialog` (or `showModalBottomSheet`) — already correct via `AppTheme.light()` component themes applied in T1.
2. Primary action: `FilledButton` (theme provides pill shape + primary color).
3. Secondary action: `OutlinedButton` or `TextButton` (theme provides outlined style).
4. Text fields: themed `InputDecoration` (pill-shaped, filled — from `AppTheme.light()`).
5. `ConfirmationDialog` destructive variant: primary action uses `FilledButton.styleFrom(backgroundColor: colorScheme.error)`.
6. Typography: via `Theme.of(context).textTheme.*` — no hardcoded sizes or weights.

## Acceptance Criteria
- [ ] **FR-8 / All 11 surfaces:** Each of the 11 popups renders with themed container (rounded corners, surface background), pill-shaped primary `FilledButton`, and themed text fields.
- [ ] **FR-8 / Destructive confirmation:** `ConfirmationDialog` destructive variant (delete record, delete source, etc.) uses error-color primary button.
- [ ] **NFR-1 / No hardcoded literals:** No `Color(0x…)` or `Colors.*` (excl. `transparent`) or numeric font sizes in any popup file.
- [ ] **NFR-2 / All save/cancel callbacks preserved:** Saving a record, transferring funds, editing a source — all produce identical side effects as before the migration. `fvm flutter test` passes.

## Technical Details
- **Approach:** Apply the theme — don't fight it. `AppTheme.light()` from T1 sets component themes for `Dialog`, `BottomSheet`, `FilledButton`, `OutlinedButton`, `TextButton`, `InputDecoration`. Most of the migration is removing inline overrides that conflict with the theme.
- **Pattern:** Read each popup file, identify all `ButtonStyle`, `TextStyle`, `Color`, `BorderRadius` literals, replace with theme references or delete the override so the theme kicks in.
- **`showModalBottomSheet` styling:** The `BottomSheetThemeData` in T1 provides the rounded top corners. The `showModalBottomSheet` call does not need `shape:` or `backgroundColor:` arguments unless there is a one-off requirement.
- **Error on incomplete popups:** If a popup has complex custom layout that resists mechanical theming, leave an `// TODO(T8): complete styling` comment and move on. The T10 verification sweep will catch remaining violations.
