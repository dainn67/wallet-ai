# T3: Shared UI primitives (IconSquare + SectionLabel)

## Context
The new design uses a "tinted rounded icon-square" pattern in at least three distinct places: category icons, transaction-row entity type indicators, and the AI chat bubble sparkle chip. Without a shared `IconSquare` widget, each surface would reimplement the same rounded container + tinted background + padded icon ‚Äî the primary source of hardcoded literal drift that NFR-1 forbids. `SectionLabel` similarly appears as section headers in multiple tabs. These primitives are thin layout widgets with no behavior ‚Äî straightforward to build, high leverage across the whole epic.

## Description
Create two new widgets in `lib/components/`: `IconSquare` (tinted rounded icon container) and `SectionLabel` (small-caps gray section header). Export both from the existing `lib/components/components.dart` barrel. All downstream tasks (T5, T6, T7, T8) import these primitives instead of reimplementing their own version.

Architecture decision in scope: AD-3 (single shared `IconSquare` primitive).

## Acceptance Criteria
- [ ] **NFR-1 / Primitive exists:** `IconSquare` widget renders a `size √ó size` rounded container with `tint` background color and centered `icon` glyph in a complementary foreground color (white or dark based on luminance).
- [ ] **NFR-1 / Token consumption:** `IconSquare` default `size` is `AppSpacing.iconSquare` (40) ‚Äî not a hardcoded `40.0`.
- [ ] **NFR-1 / SectionLabel:** `SectionLabel` renders a small-caps (`TextCapitalization.characters` or `toUpperCase()`) text in `AppColors.onSurfaceVariant` at `labelMedium` text style.
- [ ] **NFR-1 / No inline reimplementation:** No other file in `lib/` reimplements a rounded icon-square container after this task.
- [ ] **NFR-2 / No test regressions:** `fvm flutter test` passes after adding these files.

## Implementation Steps

### Step 1: Create `lib/components/icon_square.dart`
```dart
class IconSquare extends StatelessWidget {
  const IconSquare({
    super.key,
    required this.icon,
    required this.tint,
    this.size = AppSpacing.iconSquare,
  });

  final IconData icon;
  final Color tint;
  final double size;

  @override
  Widget build(BuildContext context) {
    // Foreground color: white if tint is dark, else dark neutral
    final onTint = tint.computeLuminance() > 0.4
        ? AppColors.neutral
        : AppColors.onPrimary;

    return Container(
      width: size,
      height: size,
      decoration: BoxDecoration(
        color: tint.withValues(alpha: 0.12),
        borderRadius: BorderRadius.circular(AppRadius.tile),
      ),
      child: Icon(icon, color: tint, size: size * 0.5),
    );
  }
}
```
Notes:
- Background is `tint.withValues(alpha: 0.12)` ‚Äî a translucent tint for the container; icon uses full `tint` color.
- `AppRadius.tile` (12) used for the corner radius ‚Äî matches the design mockups for icon containers.
- Import `app_theme.dart` via the `configs.dart` barrel.

### Step 2: Create `lib/components/section_label.dart`
```dart
class SectionLabel extends StatelessWidget {
  const SectionLabel(this.text, {super.key});

  final String text;

  @override
  Widget build(BuildContext context) {
    return Text(
      text.toUpperCase(),
      style: Theme.of(context).textTheme.labelSmall?.copyWith(
        color: AppColors.onSurfaceVariant,
        letterSpacing: 0.8,
        fontWeight: FontWeight.w500,
      ),
    );
  }
}
```

### Step 3: Export from `lib/components/components.dart` barrel
Add to `lib/components/components.dart`:
```dart
export 'icon_square.dart';
export 'section_label.dart';
```
Preserve all existing exports.

## Interface Contract

### Produces for T5, T6, T7, T8, T9:
- File: `lib/components/icon_square.dart`
  - Export: `IconSquare({required IconData icon, required Color tint, double size = AppSpacing.iconSquare})`
  - Guaranteed: No behavior ‚Äî pure layout. Size defaults to `AppSpacing.iconSquare`. Tint drives both background (12% opacity) and icon color.
- File: `lib/components/section_label.dart`
  - Export: `SectionLabel(String text)`
  - Guaranteed: Renders text uppercased with `labelSmall` style + `AppColors.onSurfaceVariant` color.

## Technical Details
- **Approach:** Thin stateless widgets. No state, no behavior. Direct token consumption.
- **Pattern to follow:** See `lib/components/chat_bubble.dart` for existing component structure.
- **Edge case ‚Äî luminance threshold:** The `.computeLuminance() > 0.4` threshold works for the 6 category accent colors defined in T1. If a custom tint produces wrong foreground, adjust threshold in that specific `IconSquare` call using the `tint.withValues(alpha: 1.0)` approach.
- **Do not add a `pill_chip.dart`** ‚Äî the themed `Chip` widget with `shape: StadiumBorder()` from `AppTheme.light()` is sufficient for pill chips. Only create `icon_square.dart` and `section_label.dart`.
