# T2: Plus Jakarta Sans font asset wiring

## Context
The design system requires Plus Jakarta Sans as the sole typeface. The project prohibits `google_fonts` (FOUT risk, dependency risk). Fonts must be bundled as local TTF assets in `assets/fonts/`. T1 declares `fontFamily: 'PlusJakartaSans'` in `AppTheme.light()` — this task delivers the actual font files and registers them in `pubspec.yaml`. Without this, all text falls back to the system font (silently, no exception).

## Description
Verify the 5 Plus Jakarta Sans TTF files are present in `assets/fonts/` (user confirmed they were added and committed). Replace the existing Poppins font block in `pubspec.yaml` with the PlusJakartaSans family declaration for all 5 weights. Run `fvm flutter pub get` to register. Verify airplane-mode rendering on device.

## Acceptance Criteria
- [ ] **FR-2 / Files present:** `assets/fonts/PlusJakartaSans-Regular.ttf`, `PlusJakartaSans-Italic.ttf`, `PlusJakartaSans-Medium.ttf`, `PlusJakartaSans-SemiBold.ttf`, `PlusJakartaSans-Bold.ttf` all exist.
- [ ] **FR-2 / pubspec declared:** `pubspec.yaml` has a `PlusJakartaSans` font family entry covering weights 400 (normal + italic), 500, 600, 700.
- [ ] **NFR-3 / No google_fonts:** `grep -r "google_fonts" lib/` returns empty.
- [ ] **FR-2 / Airplane-mode render:** All five weights render from local asset on device with wifi/data off (no network font fallback).
- [ ] **FR-2 / No Poppins remnant:** No Poppins `fontFamily` reference remains in `pubspec.yaml` or any Dart file under `lib/`.

## Implementation Steps

### Step 1: Verify font files in `assets/fonts/`
Run:
```bash
ls assets/fonts/PlusJakartaSans-*.ttf
```
Expected output: 5 files — `Regular`, `Italic`, `Medium`, `SemiBold`, `Bold`. If any are missing, stop and report — do not proceed.

### Step 2: Replace font block in `pubspec.yaml`
Find the existing `fonts:` block under `flutter:` that declares the Poppins family. Replace it entirely with:
```yaml
  fonts:
    - family: PlusJakartaSans
      fonts:
        - asset: assets/fonts/PlusJakartaSans-Regular.ttf
          weight: 400
        - asset: assets/fonts/PlusJakartaSans-Italic.ttf
          weight: 400
          style: italic
        - asset: assets/fonts/PlusJakartaSans-Medium.ttf
          weight: 500
        - asset: assets/fonts/PlusJakartaSans-SemiBold.ttf
          weight: 600
        - asset: assets/fonts/PlusJakartaSans-Bold.ttf
          weight: 700
```
Preserve all other sections of `pubspec.yaml` (dependencies, assets, dev_dependencies) unchanged.

### Step 3: Run `fvm flutter pub get`
Regenerates `pubspec.lock` and validates the font asset paths. Fix any YAML indentation errors surfaced by this command.

### Step 4: Audit for Poppins remnants
```bash
grep -rn "Poppins" lib/ pubspec.yaml
```
For each match: remove or replace with `'PlusJakartaSans'`.

### Step 5: Verify no google_fonts usage
```bash
grep -rn "google_fonts" lib/ pubspec.yaml
```
Expected: zero matches. If any found, remove the import and the `GoogleFonts.*` call.

## Technical Details
- **Approach:** Local asset font registration — identical to the existing Poppins setup being replaced.
- **Pattern:** `pubspec.yaml > flutter > fonts` block is the Flutter-standard mechanism.
- **Edge case:** If the `assets/fonts/` directory is not listed under `pubspec.yaml > flutter > assets`, add it. Check with `grep -A5 "assets:" pubspec.yaml`.
- **No Dart file changes needed** — T1's `fontFamily: 'PlusJakartaSans'` string matches the `family:` key exactly.
