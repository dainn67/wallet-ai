# T4: App shell chrome (AppBar + drawer + bottom nav)

## Context
The `HomeScreen` is the outermost shell wrapping all tabs. It owns the `AppBar`, `Drawer`, and `BottomNavigationBar` (or `NavigationBar`). Redesigning it first makes the new design language visible across the entire app immediately — even before individual tab interiors are migrated. This gives early visual confirmation that `AppTheme.light()` is wired correctly. The TestTab `if (kDebugMode)` gating (AD-5) also lives here and must be handled carefully to avoid breaking the `PageController` index math.

## Description
Modify `lib/screens/home/home_screen.dart` to:
1. Replace the current `AppBar` with the new design: hamburger leading icon, "Wally AI" wordmark in `AppColors.primary` SemiBold, notification bell trailing icon.
2. Switch `BottomNavigationBar` to `NavigationBar` (Material 3) with `NavigationDestination` items.
3. Wrap the TestTab `NavigationDestination` and its `PageView` page in `if (kDebugMode)` compile-out blocks (AD-5).
4. Re-skin the drawer to consume the new token system without changing its structure or navigation callbacks.

Architecture decisions in scope: AD-5 (`kDebugMode` compile-out for TestTab).

## Acceptance Criteria
- [ ] **FR-3 / AppBar design:** AppBar shows hamburger (`Icons.menu`) leading, "Wally AI" wordmark centered-left in `AppColors.primary` + `FontWeight.w600`, notification bell (`Icons.notifications_outlined`) trailing.
- [ ] **FR-3 / Drawer accessible:** Hamburger tap opens the drawer from `HomeScreen`; existing drawer items and navigation callbacks unchanged.
- [ ] **FR-4 / NavigationBar:** `BottomNavigationBar` replaced by `NavigationBar`; 3 production destinations (Chat, Records, Categories) always visible; active destination shows filled icon + label with `AppColors.primaryContainer` indicator.
- [ ] **FR-4 / TestTab gated:** In debug mode, `NavigationBar` shows 4 destinations; in release mode exactly 3. `PageController` indices are correct in both modes.
- [ ] **FR-4 / No hardcoded tab index:** No raw integer literal `3` or `2` used to refer to a tab by position — use a named constant or `_tabCount` variable derived from the list length.
- [ ] **NFR-2 / No regressions:** `fvm flutter test` passes after this task.

## Implementation Steps

### Step 1: Audit `PageController` and tab-index callers
Before touching any code, search for all places that reference tab indices:
```bash
grep -n "animateToPage\|jumpToPage\|_currentIndex\|currentPage\|== 3\|== 2\|tabIndex\|pageIndex" lib/screens/home/home_screen.dart
```
Map out the complete tab-switching logic. This is critical — changing tab count without updating all callers will cause index-out-of-bounds or wrong-tab navigation.

### Step 2: Define tab list with `kDebugMode` gating
Replace any hardcoded tab arrays with a derived list:
```dart
// Build tab destinations at construction time
final List<_TabConfig> _tabs = [
  _TabConfig(icon: Icons.chat_bubble_outline, label: 'Chat', page: const ChatTab()),
  _TabConfig(icon: Icons.receipt_long_outlined, label: 'Records', page: const RecordsTab()),
  _TabConfig(icon: Icons.category_outlined, label: 'Categories', page: const CategoriesTab()),
  if (kDebugMode)
    _TabConfig(icon: Icons.science_outlined, label: 'Test', page: const TestTab()),
];
```
Where `_TabConfig` is a private data class: `class _TabConfig { final IconData icon; final String label; final Widget page; }`.

Use `_tabs.length` everywhere a tab count is needed. Use `_tabs[index].page` for the `PageView` children.

### Step 3: Replace AppBar
```dart
AppBar(
  leading: Builder(builder: (context) => IconButton(
    icon: const Icon(Icons.menu),
    onPressed: () => Scaffold.of(context).openDrawer(),
  )),
  title: Text(
    'Wally AI',
    style: Theme.of(context).textTheme.titleLarge?.copyWith(
      color: AppColors.primary,
      fontWeight: FontWeight.w600,
    ),
  ),
  actions: [
    IconButton(
      icon: const Icon(Icons.notifications_outlined),
      onPressed: () {}, // placeholder — notification tap has no behavior in v1
    ),
    const SizedBox(width: AppSpacing.sm),
  ],
  backgroundColor: Theme.of(context).colorScheme.surface,
  elevation: 0,
  surfaceTintColor: Colors.transparent,
)
```

### Step 4: Replace `BottomNavigationBar` with `NavigationBar`
```dart
NavigationBar(
  selectedIndex: _currentIndex,
  onDestinationSelected: (index) {
    setState(() => _currentIndex = index);
    _pageController.animateToPage(index, duration: const Duration(milliseconds: 300), curve: Curves.easeInOut);
  },
  destinations: _tabs.map((t) => NavigationDestination(
    icon: Icon(t.icon),
    label: t.label,
  )).toList(),
)
```
Remove the old `BottomNavigationBar` entirely.

### Step 5: Re-skin the drawer
The drawer widget tree (its items, navigation callbacks, user info section) is unchanged. Apply token-based styling:
- Drawer background: `Theme.of(context).colorScheme.surface`
- Active tile: `ListTile` with `tileColor: AppColors.primaryContainer` when selected
- Text and icon colors: `AppColors.onSurface` / `AppColors.primary` for active
- Font via `Theme` (no manual `TextStyle` with hardcoded family or size)

## Technical Details
- **Approach:** AD-5 — `if (kDebugMode)` list literal gating (not runtime conditional). Dart eliminates the dead branch at compile time.
- **Pattern to follow:** See current `lib/screens/home/home_screen.dart` for existing drawer structure and PageController usage.
- **Critical:** `_tabs` list must be `final` and built once (in `initState` or as a field). Do not build it inside `build()` — it must be stable for `PageController.initialPage` to match.
- **Material 3 note:** `NavigationBar` requires `useMaterial3: true` in `ThemeData` (already set by T1's `AppTheme.light()`).
- **Drawer opener pattern:** Use `Builder` wrapper around the hamburger `IconButton` so `Scaffold.of(context)` finds the correct `Scaffold`. See existing Scaffold documentation pattern.
