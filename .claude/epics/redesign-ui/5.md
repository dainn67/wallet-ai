# T5: Assistant (Chat) tab redesign

## Context
The chat tab is the primary surface of Wally AI — the most user-facing part of the redesign. It has 4 component files: the tab scaffold, the chat bubble renderer, the suggested prompts bar, and the suggestion banner. The suggestion banner is the only component in the entire epic that requires a test file update — exactly 2 lines in `suggestion_banner_test.dart` must change from `find.byType(FilledButton)` to `find.byType(TextButton)`. Streaming behavior, auto-scroll logic, and the double-tap guard state machine must be preserved byte-for-byte in their behavior.

Architecture decisions in scope: AD-3 (`IconSquare` sparkle chip), AD-4 (SuggestionBanner in-place refactor).

## Description
Migrate all 4 chat-related files to the new design language:
- `chat_tab.dart` — tab background, pill input bar, send button
- `chat_bubble.dart` — assistant/user bubble variants with `IconSquare`; AI-parsed expense inline card
- `suggested_prompts_bar.dart` — horizontal scrolling pill chips
- `suggestion_banner.dart` — in-place refactor: keep class/callbacks/state machine, swap FilledButton → TextButton chips

Update `test/components/suggestion_banner_test.dart` lines 130 and 156 only (the `find.byType(FilledButton)` assertions → `find.byType(TextButton)`). No other test changes.

## Acceptance Criteria
- [ ] **FR-5 / Chat background:** `chat_tab.dart` uses `Theme.of(context).colorScheme.surfaceContainerLow` as tab scaffold background.
- [ ] **FR-5 / Input bar:** Input bar is a single pill `Container` (border radius `AppRadius.input`) containing a `+` icon button (opens camera/gallery — unchanged behavior), a `TextField`, and a circular filled primary `send` button.
- [ ] **FR-5 / Assistant bubble:** Assistant messages render as white rounded cards with a leading lavender `IconSquare(icon: Icons.auto_awesome, tint: AppColors.primary)` sparkle chip.
- [ ] **FR-5 / User bubble:** User messages render with a trailing dark avatar circle (`AppColors.neutral` background, white initials or `Icons.person`).
- [ ] **FR-5 / Expense inline card:** AI-parsed expense cards inside the assistant bubble use `SectionLabel('EXPENSE DETECTED')` + `IconSquare` for the category + bold amount in `AppColors.primary`.
- [ ] **FR-5 / Suggested prompts:** Prompts bar renders as horizontal-scrolling outlined gray pill `ActionChip`s (using themed `Chip`).
- [ ] **FR-5 / SuggestionBanner class preserved:** `find.byType(SuggestionBanner)` still resolves the widget — class name unchanged.
- [ ] **FR-5 / SuggestionBanner internals:** `SuggestionBanner` renders italic confirmation text + two `TextButton` chips (not `FilledButton`). Double-tap guard state machine preserved — rapid double-taps do not fire `onConfirm` twice.
- [ ] **NFR-2 / Only 2 test lines changed:** `suggestion_banner_test.dart` has exactly 2 changed lines (lines 130 and 156: `FilledButton` → `TextButton`). No other test file is modified.
- [ ] **NFR-2 / Streaming preserved:** `isStreaming`, `isAnalyzing`, auto-scroll (`jumpTo`) behavior is untouched — verify by tracing callers in `chat_tab.dart` after the edit.

## Implementation Steps

### Step 1: Migrate `lib/components/suggestion_banner.dart` (highest risk — do first)
This is the most sensitive file. Read the current implementation carefully, then:
1. Keep the widget class `SuggestionBanner` with all existing constructor parameters: `onConfirm`, `onCancel`, and any `suggestion`/`category` display fields.
2. Keep the double-tap guard: the existing `_confirmCount` counter or `_lastTapTime` debounce state machine — preserve it exactly.
3. Replace the outer `Card` container with `Padding(padding: EdgeInsets.symmetric(horizontal: AppSpacing.lg, vertical: AppSpacing.sm))`.
4. Replace the `FilledButton` confirm action with `TextButton` styled as a chip:
   ```dart
   TextButton(
     onPressed: _guardedOnConfirm,
     style: TextButton.styleFrom(
       backgroundColor: AppColors.primaryContainer,
       foregroundColor: AppColors.primary,
       shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(AppRadius.chip)),
       padding: EdgeInsets.symmetric(horizontal: AppSpacing.md, vertical: AppSpacing.xs),
     ),
     child: Text(/* confirm label */),
   )
   ```
5. Replace the cancel `FilledButton`/`TextButton` with `TextButton`:
   ```dart

     onPressed: widget.onCancel,

       foregroundColor: AppColors.onSurfaceVariant,

6. Add the suggestion text in italic:

   )
   ```

## Technical Details
- **Approach:** AD-4 (SuggestionBanner in-place). The widget class name, its constructor signature, and its state machine are identity-preserved.
- **Pattern to follow:** `lib/components/chat_bubble.dart` existing structure — reading it before editing is mandatory.
- **Double-tap guard:** The guard is state in `_SuggestionBannerState`. Do not convert `SuggestionBanner` from `StatefulWidget` to `StatelessWidget` — the guard requires mutable state.
- **`fontStyle: FontStyle.italic`:** The suggestion text italic requires weight 400i — which T2 registers in `pubspec.yaml` as `style: italic` for weight 400. If T2 hasn't landed, the italic will synthesize (acceptable during development).
- **Edge case — long suggestion text:** The inline italic text + two `TextButton` chips must not overflow. Use `Wrap` or a `Column` layout if single `Row` overflows on narrow screens.
