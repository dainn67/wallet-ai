# T7: Categories tab redesign

## Context
The Categories tab has two files: the tab scaffold (date-period navigation, "Add Category" button, list composition) and the `category_widget.dart` (parent row with expand/collapse, sub-rows). The InkWell absorption pattern ‚Äî where setting `onTap` non-null on an `ExpansionTile` child absorbs taps and prevents tile expansion ‚Äî is a documented subtle interaction that must be preserved. NTH-2 (per-category accent assignment) is deferred: all parent category `IconSquare`s use `AppColors.primary` as the tint in v1; the 6-step ramp assignment is a follow-up PRD.

## Description
Migrate `categories_tab.dart` (date-period pill, add-category button, tab scaffold) and `category_widget.dart` (parent row card, sub-row card with left accent border). All expand/collapse, edit, add-sub-category, and drill-down behaviors are unchanged. No logic touches.

## Acceptance Criteria
- [ ] **FR-7 / Date-period pill:** Top of the tab has a `< MMM yyyy >` date navigator rendered as a `Container` with `AppColors.primaryContainer` background, `AppRadius.pill` border radius, `< Previous` and `Next >` icon buttons.
- [ ] **FR-7 / Add button:** "Add Category" rendered as a `FilledButton` (pill shape, primary color) using the themed `FilledButton` style.
- [ ] **FR-7 / Parent category row:** White `Card` (themed) with leading `IconSquare(tint: AppColors.primary)` + category icon, bold category name, monthly total, expand chevron, and pencil edit icon.
- [ ] **FR-7 / Sub-category row:** White card with a 3dp left border in `AppColors.primary` (via `BoxDecoration(border: Border(left: BorderSide(color: AppColors.primary, width: 3)))`), sub-category name + total, pencil edit. No `IconSquare`.
- [ ] **FR-7 / InkWell absorption preserved:** Tapping the edit pencil icon on a parent row does NOT expand/collapse the tile ‚Äî edit action takes precedence.
- [ ] **NFR-1 / No hardcoded literals:** No inline `Color(0x‚Ä¶)` or numeric radii in `category_widget.dart`.
- [ ] **NFR-2 / Behavior parity:** Expand/collapse, add category, add sub-category, edit category, drill-down to `CategoryRecordsBottomSheet` ‚Äî all unchanged. `fvm flutter test` passes.

### Step 2: Migrate `lib/components/category_widget.dart`

**Read the current implementation carefully.** Identify: where `ExpansionTile` is used, where the edit `IconButton` is placed, where sub-rows are rendered, and the InkWell absorption pattern.

**Parent row card:**
```dart
Card(
  child: Column(children: [
    ListTile(
      leading: IconSquare(
        icon: category.icon ?? Icons.category_outlined,
        tint: AppColors.primary,
      ),
      title: Text(category.name, style: Theme.of(context).textTheme.bodyLarge?.copyWith(fontWeight: FontWeight.w600)),
      trailing: Row(mainAxisSize: MainAxisSize.min, children: [
        Text(formattedTotal, style: Theme.of(context).textTheme.bodyMedium?.copyWith(color: AppColors.onSurfaceVariant)),
        const SizedBox(width: AppSpacing.xs),
        IconButton(
          icon: const Icon(Icons.edit_outlined, size: 18),
          onPressed: () => _onEditCategory(category), // existing callback
          // InkWell absorption: onPressed non-null absorbs tap here, preventing ExpansionTile from firing
        ),
        RotationTransition(
          turns: _expandAnimation,
          child: const Icon(Icons.expand_more_rounded),
        ),
      ]),
      onTap: _onToggleExpand, // existing expand logic
    ),
    AnimatedCrossFade(/* sub-rows when expanded */),
  ]),
)
```
**Key:** Do not replace `ListTile.onTap` with an `InkWell` wrapper if the existing code uses `ListTile.onTap` for expansion ‚Äî that pattern must stay as-is.

**Sub-category row:**
```dart
Container(
  margin: const EdgeInsets.only(top: 1),
  decoration: BoxDecoration(
    color: Theme.of(context).colorScheme.surface,
    border: const Border(left: BorderSide(color: AppColors.primary, width: 3)),
    borderRadius: BorderRadius.only(
      bottomLeft: Radius.circular(AppRadius.tile),
      bottomRight: Radius.circular(AppRadius.tile),
    ),
  ),
  child: ListTile(
    title: Text(sub.name),
    trailing: Row(mainAxisSize: MainAxisSize.min, children: [
      Text(formattedSubTotal),
      const SizedBox(width: AppSpacing.xs),
      IconButton(icon: const Icon(Icons.edit_outlined, size: 18), onPressed: () => _onEditSub(sub)),
    ]),
    onTap: () => _onDrillDown(sub), // existing drill-down
  ),
)
```
No `IconSquare` on sub-rows ‚Äî matches design spec.

## Technical Details
- **Approach:** Structural migration ‚Äî both files get rewritten builds but zero behavior changes.
- **Pattern to follow:** Read `lib/components/category_widget.dart` current implementation. The InkWell absorption behavior is documented in `project_context/architecture.md` ‚Äî read it before touching the expand/collapse interaction.
- **InkWell absorption:** If the existing code uses `ListTile(onTap: _expand)` with a trailing `IconButton(onPressed: _edit)`, this already absorbs correctly ‚Äî `IconButton` captures the tap before `ListTile.onTap` fires. Do not convert to a plain `GestureDetector` or `InkWell` wrapper that would break this.
- **NTH-2 deferred:** All parent `IconSquare` tints are `AppColors.primary` in v1. No per-category color assignment. Add a `// NTH-2: per-category tint deferred` comment at the `IconSquare` call site so future implementers can locate it.
- **`border: Border(left: ...)` on sub-rows:** The left border inside a `BoxDecoration` does not interact with `ListTile`'s ink splash. No clip needed ‚Äî the splash is contained within the `ListTile` bounds naturally.
