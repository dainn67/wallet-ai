# T6: Records tab redesign

## Context
The Records tab is the second-most-used surface after Chat. It has a 3-zone overview card at the top (net worth / income-expense / money-source row), then a scrolling list of dated record rows. Balance-mask toggle, edit/delete callbacks, and sort order are behavior — all must be preserved unchanged. The visual migration replaces stock Material 3 defaults with the new token-driven design including `IconSquare` type indicators and `AppSemanticColors` income/expense/transfer tints.

## Description
Migrate 4 files: `records_tab.dart` (scaffold), `records_overview.dart` (3-zone overview card), `record_widget.dart` (individual record row), `date_divider.dart` (date section header). The overview card is a substantial structural rewrite — it needs to go from a single card to a 3-zone layout. Record rows get `IconSquare` tinting by transaction type. Date dividers use `SectionLabel`.

**Money-source accent rule (PRD warning W2 resolution):** Each money-source card in Zone 3 uses a color from `AppSemanticColors.categoryAccents[idx % 6]` — sequential index modulo 6. Document this choice in the PR description.

## Acceptance Criteria
- [ ] **FR-6 / Zone 1 — Net Worth hero:** `RecordsOverview` Zone 1 shows a large net worth amount (masked when balance-mask is on); mask toggle button visible.
- [ ] **FR-6 / Zone 2 — Income/Expense tiles:** Two side-by-side tiles with `AppSemanticColors.incomeGreen` and `AppSemanticColors.expenseRed` tint backgrounds (12% opacity fill) and contrasting text.
- [ ] **FR-6 / Zone 3 — Money-source cards:** Horizontal scrolling row of money-source cards; each card background tint is `categoryAccents[idx % 6].withValues(alpha: 0.12)`.
- [ ] **FR-6 / Record row — IconSquare:** Each `RecordWidget` row has a leading `IconSquare` with: `income → AppSemanticColors.incomeGreen`, `expense → AppSemanticColors.expenseRed`, `transfer → AppSemanticColors.transferTint`.
- [ ] **FR-6 / Record row — amount color:** Income amounts are `AppSemanticColors.incomeGreen`; expense amounts are `AppSemanticColors.expenseRed`; transfer amounts are `AppColors.onSurface`.
- [ ] **FR-6 / Date divider:** `DateDivider` uses `SectionLabel(dateString)` — no more custom `Text` with hardcoded style.
- [ ] **NFR-1 / No hardcoded literals:** No `Color(0x…)`, `Colors.*` (excl. `transparent`), numeric `fontSize`, or numeric `BorderRadius` in the 4 files.
- [ ] **NFR-2 / Behavior parity:** Balance-mask toggle masks net worth + income; record edit/delete callbacks fire correctly; sort order unchanged; `fvm flutter test` passes.

## Implementation Steps

### Step 1: Migrate `lib/components/date_divider.dart`
Simplest change. Find the existing `DateDivider` or `_DateHeader` widget and replace its `Text` rendering with:
```dart
SectionLabel(dateLabel)
```
Where `dateLabel` is the formatted date string already computed. Import `SectionLabel` from `lib/components/components.dart`.

### Step 2: Migrate `lib/components/record_widget.dart`
Read the current widget's `build` method, then:
1. Add `_iconForRecord(Record r)` helper:
   ```dart
   IconData _iconForRecord(Record r) => switch (r.type) {
     'income'   => Icons.arrow_downward_rounded,
     'expense'  => Icons.arrow_upward_rounded,
     'transfer' => Icons.swap_horiz_rounded,
     _          => Icons.receipt_long_outlined,
   };
   ```
2. Add `_tintForRecord(Record r, AppSemanticColors sem)` helper:
   ```dart
   Color _tintForRecord(Record r, AppSemanticColors sem) => switch (r.type) {
     'income'   => sem.incomeGreen,
     'expense'  => sem.expenseRed,
     'transfer' => sem.transferTint,
     _          => AppColors.primary,
   };
   ```
3. Access `AppSemanticColors` in `build`:
   ```dart
   final sem = Theme.of(context).extension<AppSemanticColors>()!;
   ```
4. Replace the existing leading icon/container with:
   ```dart
   IconSquare(icon: _iconForRecord(widget.record), tint: _tintForRecord(widget.record, sem))
   ```
5. Color the amount `Text`:
   ```dart
   Text(
     formattedAmount,
     style: Theme.of(context).textTheme.bodyMedium?.copyWith(
       color: widget.record.type == 'income' ? sem.incomeGreen
            : widget.record.type == 'expense' ? sem.expenseRed
            : Theme.of(context).colorScheme.onSurface,
       fontWeight: FontWeight.w600,
     ),
   )
   ```
6. Edit/delete pencil icon and callbacks — **do not touch**.

### Step 3: Rebuild `lib/components/records_overview.dart`
The current `RecordsOverview` is likely a single-card layout. Rewrite the `build` method to produce a 3-zone column:

**Zone 1 — Net Worth hero:**
```dart
Container(
  width: double.infinity,
  padding: const EdgeInsets.all(AppSpacing.xl),
  decoration: BoxDecoration(
    color: Theme.of(context).colorScheme.surface,
    borderRadius: BorderRadius.circular(AppRadius.card),
    boxShadow: [BoxShadow(color: Colors.black.withValues(alpha: 0.04), blurRadius: 8, offset: const Offset(0, 2))],
  ),
  child: Column(crossAxisAlignment: CrossAxisAlignment.start, children: [
    SectionLabel('Net Worth'),
    const SizedBox(height: AppSpacing.sm),
    Row(children: [
      Text(
        widget.isMasked ? '••••••' : widget.formattedNetWorth,
        style: Theme.of(context).textTheme.headlineMedium?.copyWith(fontWeight: FontWeight.w700),
      ),
      const Spacer(),
      IconButton(
        icon: Icon(widget.isMasked ? Icons.visibility_off_outlined : Icons.visibility_outlined),
        onPressed: widget.onToggleMask,
      ),
    ]),
  ]),
)
```

**Zone 2 — Income/Expense tiles:**
```dart
Row(children: [
  Expanded(child: _OverviewTile(
    label: 'Income',
    amount: widget.formattedIncome,
    tint: sem.incomeGreen,
    isMasked: widget.isMasked,
  )),
  const SizedBox(width: AppSpacing.sm),
  Expanded(child: _OverviewTile(
    label: 'Expenses',
    amount: widget.formattedExpense,
    tint: sem.expenseRed,
    isMasked: false, // expense is always visible per existing spec
  )),
])
```
Where `_OverviewTile` is a private widget:
```dart
class _OverviewTile extends StatelessWidget {
  // Container with tint.withValues(alpha:0.12) background, label + amount
}
```

**Zone 3 — Money-source scroll:**
```dart
SizedBox(
  height: 80,
  child: ListView.separated(
    scrollDirection: Axis.horizontal,
    itemCount: widget.moneySources.length,
    separatorBuilder: (_, __) => const SizedBox(width: AppSpacing.sm),
    itemBuilder: (context, idx) {
      final source = widget.moneySources[idx];
      final accent = sem.categoryAccents[idx % sem.categoryAccents.length];
      return _SourceCard(source: source, accent: accent);
    },
  ),
)
```

Assemble the three zones inside a `Column` with `AppSpacing.md` gaps.

## Technical Details
- **Approach:** `RecordsOverview` is a structural rebuild — the interface (constructor parameters) stays the same if possible; internal layout is replaced.
- **Balance-mask behavior:** The mask toggle (`isMasked` bool + `onToggleMask` callback) is plumbed from the parent. Do not change the toggle logic — only the visual representation of the masked state.
- **Pattern to follow:** Read `lib/components/records_overview.dart` current structure before replacing. Preserve constructor signature to avoid breaking `records_tab.dart` caller.
- **`categoryAccents` length:** The list has 6 entries. `idx % 6` is safe. Use `sem.categoryAccents.length` in the modulo to be defensive.
- **`Colors.black` in Zone 1 shadow:** `Colors.black.withValues(alpha: 0.04)` is acceptable — it's the shadow color, not a design token color. `Colors.transparent` and shadow helpers are in the NFR-1 exclusion list.
