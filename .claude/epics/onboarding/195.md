# Task: Build OnboardingDialog widget

## Context

Wally AI needs a first-launch onboarding dialog. This task builds the entire dialog as a self-contained widget with three sequential slides, a Next/Got it button, and full back-gesture blocking. It depends on Task 001 (storage key constant + l10n keys + placeholder PNG assets). Task 003 will wire this widget into `HomeScreen`.

## Description

Create `lib/components/popups/onboarding_dialog.dart` â€” a `StatefulWidget` that renders a non-dismissible `Dialog` with a `PageView` (swipe blocked) showing three configurable slides. Each slide displays an `Image.asset` above a localized `Text`. Navigation is button-only: "Next" on slides 0 and 1, "Got it" on slide 2. Tapping "Got it" writes `onboarding_complete = true` via `StorageService` then closes the dialog via `Navigator.pop`. A `PopScope(canPop: false)` wrapper blocks Android back button and iOS swipe-back. Slides are configured via a private `const` list at the top of the file. Export the widget from `lib/components/components.dart`. Write smoke tests covering all FR-2 and NFR-2 scenarios.

## Acceptance Criteria

- [ ] **FR-2 / Advancing slides:** Tapping "Next" on slide 0 â†’ dialog shows slide 1. Tapping "Next" on slide 1 â†’ dialog shows slide 2.
- [ ] **FR-2 / Got it only on last slide:** "Got it" button is present only on slide 2; "Next" button is present only on slides 0 and 1.
- [ ] **FR-2 / Blocked dismissal (tap outside):** `showDialog` called with `barrierDismissible: false` â†’ tapping outside the dialog has no effect.
- [ ] **FR-2 / Blocked dismissal (back button):** `PopScope(canPop: false)` blocks Android back; smoke test triggers `Navigator.maybePop` and asserts dialog remains.
- [ ] **FR-2 / Swipe blocked:** PageView with `NeverScrollableScrollPhysics` â†’ swipe gesture does not change slide.
- [ ] **FR-3 / Slide content displayed:** Each slide renders `Image.asset(slide.imageAsset)` above `Text(l10n.translate(slide.textKey))`; the correct image and text appear for each slide index.
- [ ] **FR-4 / Completion flag written:** After "Got it" tap, `StorageService().getBool(StorageService.keyOnboardingComplete)` returns `true`.
- [ ] **NFR-2 / Back-button lock:** Both Android back button and iOS swipe-back produce no dismissal on any slide (covered by `PopScope(canPop: false)` + smoke test).

### Step 1: Create `onboarding_dialog.dart`

Create `lib/components/popups/onboarding_dialog.dart` (following the style of the existing `lib/components/popups/category_records_bottom_sheet.dart`).

```dart
// Slide data class
class _OnboardingSlide {
  final String imageAsset;
  final String textKey;  // l10n key
  const _OnboardingSlide({required this.imageAsset, required this.textKey});
}

// Slide config â€” only this list needs changing if slides are added/reordered
const _slides = <_OnboardingSlide>[
  _OnboardingSlide(imageAsset: 'assets/onboarding/slide_1.png', textKey: 'onboarding_slide_1_text'),
  _OnboardingSlide(imageAsset: 'assets/onboarding/slide_2.png', textKey: 'onboarding_slide_2_text'),
  _OnboardingSlide(imageAsset: 'assets/onboarding/slide_3.png', textKey: 'onboarding_slide_3_text'),
];
```

Widget class `OnboardingDialog extends StatefulWidget`:
- State holds: `PageController _pageController` (initialPage: 0), `int _currentPage = 0`.
- `initState`: initialize `_pageController`.
- `dispose`: dispose `_pageController`.

Build method structure:
```dart
@override
Widget build(BuildContext context) {
  final l10n = context.watch<LocaleProvider>();
  final isLastPage = _currentPage == _slides.length - 1;

  return PopScope(
    canPop: false,
    child: Dialog(
      shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(16)),
      child: Column(
        mainAxisSize: MainAxisSize.min,
        children: [
          // PageView â€” fixed height, swipe blocked
          SizedBox(
            height: MediaQuery.of(context).size.height * 0.5,
            child: PageView.builder(
              controller: _pageController,
              physics: const NeverScrollableScrollPhysics(),
              onPageChanged: (i) => setState(() => _currentPage = i),
              itemCount: _slides.length,
              itemBuilder: (_, i) => _buildSlide(_slides[i], l10n),
            ),
          ),
          // Button row
          Padding(
            padding: const EdgeInsets.all(16),
            child: SizedBox(
              width: double.infinity,
              child: ElevatedButton(
                onPressed: isLastPage ? () => _handleGotIt(context) : _handleNext,
                child: Text(isLastPage
                    ? l10n.translate('onboarding_got_it')
                    : l10n.translate('onboarding_next')),
              ),
            ),
          ),
        ],
      ),
    ),
  );
}
```

`_buildSlide` method:
```dart
Widget _buildSlide(_OnboardingSlide slide, LocaleProvider l10n) {
  return Column(
    children: [
      Expanded(
        child: Image.asset(slide.imageAsset, fit: BoxFit.contain),
      ),
      Padding(
        padding: const EdgeInsets.symmetric(horizontal: 16, vertical: 12),
        child: Text(
          l10n.translate(slide.textKey),
          textAlign: TextAlign.center,
          style: const TextStyle(fontSize: 15),
        ),
      ),
    ],
  );
}
```

`_handleNext` method:
```dart
void _handleNext() {
  _pageController.nextPage(
    duration: const Duration(milliseconds: 300),
    curve: Curves.easeInOut,
  );
}
```

`_handleGotIt` method:
```dart
Future<void> _handleGotIt(BuildContext context) async {
  await StorageService().setBool(StorageService.keyOnboardingComplete, true);
  if (context.mounted) Navigator.of(context).pop();
}
```

Static `show` helper (entry point used by `HomeScreen`):
```dart
static Future<void> show(BuildContext context) {
  return showDialog<void>(
    context: context,
    barrierDismissible: false,
    builder: (_) => const OnboardingDialog(),
  );
}
```

## Technical Details

- **Back-button blocking:** Flutter â‰Ą3.12 uses `PopScope(canPop: false)`. The project is on Flutter 3.35.7 (per `.fvmrc`) â€” `PopScope` is available. Do NOT use the deprecated `WillPopScope`.
- **iOS swipe-back:** `showDialog` pushes a `DialogRoute`. In Flutter 3.35.7, `PopScope(canPop: false)` also intercepts the iOS back swipe on a dialog route. If testing on iOS simulator, verify the back gesture is blocked.
- **PageView size:** Use a fixed `SizedBox` height (50% of screen height) for the PageView so the button row is always visible without scrolling. Adjust during QA if the placeholder images look clipped.
- **`context.mounted` guard:** `_handleGotIt` is `async` (awaits `setBool`). Always check `context.mounted` before calling `Navigator.of(context).pop()` to avoid unmounted widget errors.
- **Pattern reference:** `lib/components/popups/category_records_bottom_sheet.dart` for file layout and `Consumer<LocaleProvider>` usage.
- **Edge case:** If `StorageService().setBool` fails (returns `false`), still dismiss the dialog â€” per PRD Risk #2, the worst case is the user sees the dialog once more on the next launch.
