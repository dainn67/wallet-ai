# Task: Wire OnboardingDialog into HomeScreen and add integration test

## Context

The `OnboardingDialog` widget (built in Task 002) exists but is not yet shown anywhere. This task wires it into `HomeScreen.initState()` via a post-frame callback that checks the SharedPreferences flag, and adds an integration test verifying that the dialog appears on first launch and is suppressed on subsequent ones. This is the final implementation task before verification.

## Description

In `lib/screens/home/home_screen.dart`, add a `WidgetsBinding.instance.addPostFrameCallback` at the end of `initState()` that reads `StorageService().getBool(StorageService.keyOnboardingComplete)` and calls `OnboardingDialog.show(context)` if the flag is not `true`. Guard with `if (!mounted) return;`. Write an integration test in `tests/integration/epic_onboarding/first_launch_gate_test.dart` covering both the first-launch (dialog shown) and returning-launch (no dialog) cases.

## Acceptance Criteria

- [ ] **FR-1 / First launch:** App opens with empty SharedPreferences → `OnboardingDialog` is shown before any tab content is interactable.
- [ ] **FR-1 / Returning launch:** App opens with `onboarding_complete = true` in SharedPreferences → `OnboardingDialog` is NOT shown; home screen loads normally.
- [ ] **FR-4 / Flag persists:** After "Got it" (written in Task 002), the next launch correctly reads `true` and skips the dialog.
- [ ] **No regression:** Existing `HomeScreen` behaviour (TabController, HomeWidget listener, dev mode) is unaffected. Full test suite passes.

## Implementation Steps

### Step 1: Add post-frame callback in `HomeScreen.initState()`

- Open `lib/screens/home/home_screen.dart`.
- Locate `initState()` (around line 31). The method currently sets up `_tabController` and registers the `HomeWidget` listeners.
- At the END of `initState()` (after all existing setup), add:
  ```dart
  WidgetsBinding.instance.addPostFrameCallback((_) {
    if (!mounted) return;

      OnboardingDialog.show(context);
    }
  });
  ```

## Technical Details

- **Why post-frame callback?** `showDialog` requires a valid `Navigator` context. The Navigator is only available after the first frame is rendered. Calling `showDialog` directly in `initState` (before the frame) would throw `Navigator operation requested with a context that does not include a Navigator`. The post-frame callback guarantees the Navigator tree is mounted.
- **Placement:** Add the callback at the END of `initState`, after `_tabController` setup and `HomeWidget` listener registration, so all other setup completes first and there is no timing conflict.
- **`mounted` guard:** The callback fires after the frame; in tests, the widget may be disposed before that. The `if (!mounted) return;` guard prevents calling `context` on a disposed widget.
- **`StorageService` init:** `StorageService.init()` is already called in `main()` before `runApp`, so it is always ready when `HomeScreen.initState` runs in production. In tests, call `await StorageService.init()` (with mock initial values set) before pumping the widget.
- **Pattern reference:** `lib/screens/home/home_screen.dart` existing `initState` for placement; `test/screens/records_tab_test.dart` for the mock provider pattern used in integration tests.
- **Edge case:** If `StorageService()._prefs` is null (init not called), `getBool` returns `null`. The check `!= true` handles `null` correctly — dialog will show. This is safe.
