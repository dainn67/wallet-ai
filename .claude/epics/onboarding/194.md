# Task: Foundation ŌĆö storage key, l10n strings, and slide assets

## Context

Wally AI has no onboarding flow. This is the first task in that epic. Before the dialog widget can be built, three things must exist: (1) a SharedPreferences key constant so the dialog can read/write its completion flag, (2) localized strings for slide text and button labels, and (3) placeholder image assets that the dialog will render on each slide. Nothing in the app yet references `onboarding_complete` ŌĆö verified by convention; a grep check is included below.

## Description

Add the `onboarding_complete` boolean key to the existing `StorageService` singleton alongside the other key constants (`keyCurrency`, `keyUserPattern`). Add five localization keys to `l10n_config.dart` for both English and Vietnamese: three slide texts and two button labels ("Next", "Got it"). Create an `assets/onboarding/` directory with three solid-colour 360├Ś640 PNG placeholder images (`slide_1.png`, `slide_2.png`, `slide_3.png`) and declare the directory in `pubspec.yaml`. Run `fvm flutter pub get` to confirm the asset pipeline loads cleanly.

## Technical Details

- **Pattern:** `StorageService` follows the static-key pattern ŌĆö all keys are `static const String` fields on the class, accessed as `StorageService.keyXxx`. Follow this exactly; don't use raw strings in call sites.
- **L10n pattern:** `lib/configs/l10n_config.dart` uses a `Map<String, Map<String, String>>` structure accessed via `LocaleProvider.translate(key)`. New entries follow alphabetical grouping within the map where practical.
- **Asset pipeline:** Flutter requires either a specific file path or a trailing-slash directory wildcard. Use `- assets/onboarding/` (trailing slash) to include all files in the directory ŌĆö this means the designer can add more slides later without touching `pubspec.yaml`.
- **Edge case:** If `l10n_config.dart` is structured differently than expected (e.g., split files), locate the `translate` method to find where strings are stored.
