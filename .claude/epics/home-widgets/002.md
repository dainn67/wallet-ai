## Context

The current `AppWidget.kt` has a tangled `when { ... }` branch in `provideContent` and a `QuickRecordBar` that conflates "isCompact" with size logic. To support the new per-breakpoint matrix (FR-1) + write/camera icon row (FR-2, FR-3) without re-introducing the same tangle, this task rewrites the composables with a clean per-breakpoint dispatch and a separate `ActionIconRow`.

## Description

Full rewrite of the Glance composable body of `AppWidget.kt`. Keep `SizeMode.Responsive` and the existing 5 `DpSize` constants (SMALL/TALL/WIDE/MEDIUM/LARGE). Replace the body with:

1. A single dispatch function that picks one composable per breakpoint based on `LocalSize.current`.
2. A redesigned `QuickRecordBar(iconOnly: Boolean)` â€” `iconOnly` is derived from `size.width <= 80.dp` so both 1Ă—1 SMALL and 1Ă—2 TALL render icon-only (per FR-1 update).
3. A new `ActionIconRow()` composable rendering two icons (write + camera) side-by-side, each with its own `clickable(actionStartActivity<MainActivity>(context, Uri.parse(...)))`.
4. Root `Box` retains its own `clickable` for FR-5 fallback. Critical: inner clickables must be applied **before** the root `clickable` so Glance's hit-test resolves the innermost first.

## Implementation Steps

### Step 1: Replace dispatch with a clean per-breakpoint switch
- Modify `android/app/src/main/kotlin/com/example/wallet_ai/AppWidget.kt`
- Inside `provideContent { ... }`, replace the current `when { ... }` branch with a function that maps each `LocalSize.current` to one of: `SmallLayout`, `TallLayout`, `WideLayout`, `MediumLayout`, `LargeLayout`.
- Each layout composable is responsible only for its own arrangement â€” no nested `when`s.

### Step 2: Refactor QuickRecordBar
- Replace the current `QuickRecordBar(..., isCompact: Boolean = false)` signature with `QuickRecordBar(context, surfaceColor, accentColor, textColor, iconOnly: Boolean)`.
- Internally, when `iconOnly = true`, render only the edit icon (no text, narrower width).
- When `iconOnly = false`, render the icon + "Add record" label as today.
- Always wire `clickable(actionStartActivity<MainActivity>(context, Uri.parse("homeWidget://record")))`.

### Step 3: Add ActionIconRow composable
- New private composable `ActionIconRow(context, surfaceColor, accentColor)` in the same file.
- Renders two circular `Box`es side-by-side, each 44 dp Ă— 44 dp with 22 dp corner radius and surface background.
- Left box: write icon (use the drawable identified in T001 â€” likely `android.R.drawable.ic_menu_edit`), clickable â†’ `homeWidget://record`.
- Right box: camera icon (use the drawable from T001 â€” `android.R.drawable.ic_menu_camera` OR bundled vector), clickable â†’ `homeWidget://camera`.
- `Spacer(GlanceModifier.width(8.dp))` between them.

### Step 4: Wire ActionIconRow into MediumLayout and LargeLayout only
- In `MediumLayout`: render after the stats row, before `QuickRecordBar`. The bar stays as a full-width affordance below the icon row.
- In `LargeLayout`: same arrangement.
- Do NOT include ActionIconRow in SmallLayout, TallLayout, or WideLayout â€” FR-2 and FR-3 explicitly scope to 2Ă—2+.

### Step 5: Root clickable ordering
- Wrap the outermost `Box` with `.clickable(actionStartActivity<MainActivity>(context, Uri.parse("homeWidget://open")))` (or empty URI â†’ MainActivity default route).
- Critical: apply this modifier on the root container only AFTER inner composables are declared. Glance's hit-test traverses from leaf to root; inner clickables on bar/icons must override.
