# Task 011: ChatApiService — add audioBase64 parameter and top-level audio field

## Context

The AI chat request body currently supports `query` (text) and `images` (array of base64 JPEG strings). Voice input requires a new `audio` field containing a base64-encoded AAC file. The server uses the presence of this field to route the request to the voice-aware Gemini prompt. This task extends `ChatApiService.streamChat()` to accept the audio and pack it correctly into the request body — mirroring exactly how `images` was added in the image-input epic (AD-2 from the voice-input epic).

## Acceptance Criteria

- [ ] **FR-4 / Voice-only message:** `streamChat(query: '', audioBase64: '<non-empty>')` builds a request body where `inputs['audio']` equals the provided string and `inputs['query']` is `''`.
- [ ] **FR-4 / Voice + text:** `streamChat(query: 'lunch', audioBase64: '<non-empty>')` includes both `inputs['query'] = 'lunch'` and `inputs['audio'] = '<non-empty>'`.
- [ ] **FR-4 / No audio:** `streamChat(query: 'hello', audioBase64: null)` builds a request body with NO `audio` key at all (key must be absent, not null, not empty string).
- [ ] **FR-4 / Empty string guard:** `streamChat(query: 'hello', audioBase64: '')` treats empty string the same as null — no `audio` key in body.
- [ ] **Existing tests:** All pre-existing `ChatApiService` tests continue to pass unchanged.

## Technical Details

- **Pattern (AD-2):** Exact mirror of how `images` was added. The constraint "omit key entirely when absent" is important because the server checks for the key's presence to decide routing — `null` or `""` would be ambiguous.
- **No breaking change:** The new parameter is optional with a default of `null`. All existing call sites (`ChatProvider._handleStream`) will continue to compile and behave identically.
- **No interaction with `images`:** The two fields are independent at this layer. The constraint that `audio` and `images` are mutually exclusive is enforced at the provider layer (task 012) — this service does not need to validate that.
- **`query` may be empty:** The existing `query: message` assignment is unchanged. An empty `message` string is valid and occurs in the voice-only scenario.
