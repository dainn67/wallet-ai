# Task 013: Error strings — voice and image failure messages

## Context

PRD FR-5 requires two distinct error messages: one for when the AI cannot interpret a voice recording ("I didn't catch that. Please try again.") and one for when an attached image fails to load ("Couldn't load the image. Please try again."). This task adds both localized strings to the app's string configuration so that tasks 010 and 012 can reference them via `_localeProvider?.translate(key)`. It also audits the existing image processing error paths to surface the image error message visually — a retroactive fix to the image-input feature that was shipped without an error snackbar.

## Acceptance Criteria

- [ ] **FR-5 / Voice string EN:** `l10n_config.dart` contains key `voice_didnt_catch_that` → `"I didn't catch that. Please try again."`
- [ ] **FR-5 / Voice string VN:** Same key → `"Mình chưa nghe rõ. Bạn thử lại nhé."` (or equivalent Vietnamese).
- [ ] **FR-5 / Image string EN:** Key `image_load_failed` → `"Couldn't load the image. Please try again."`
- [ ] **FR-5 / Image string VN:** Same key → `"Không tải được ảnh. Bạn thử lại nhé."` (or equivalent Vietnamese).
- [ ] **FR-5 / Image error surfaced:** When image processing or compression fails during a send (e.g., the picked file is corrupted or oversize after compression), a `SnackBar` containing the `image_load_failed` string appears; no silent failure, no empty record card.
- [ ] **Existing strings unchanged:** All pre-existing keys in `l10n_config.dart` are unmodified.

## Implementation Steps

### Step 1: Locate the string map in lib/configs/l10n_config.dart

- Open `lib/configs/l10n_config.dart`.
- Find the structure that maps language → key → string. It is likely a `Map<String, Map<String, String>>` or a `translate(key)` method with embedded maps.
- Identify the English and Vietnamese string blocks.

### Step 2: Add voice error string

- Add to the **English** block:
  ```
  'voice_didnt_catch_that': "I didn't catch that. Please try again.",
  ```
- Add to the **Vietnamese** block:
  ```
  'voice_didnt_catch_that': "Mình chưa nghe rõ. Bạn thử lại nhé.",
  ```

### Step 3: Add image error string

- Add to the **English** block:
  ```
  'image_load_failed': "Couldn't load the image. Please try again.",
  ```
- Add to the **Vietnamese** block:
  ```
  'image_load_failed': "Không tải được ảnh. Bạn thử lại nhé.",
  ```

### Step 4: Surface image error in the image pipeline

- In `lib/screens/home/tabs/chat_tab.dart`, find the places where image files are picked and compressed before sending. The existing `ImagePickerService` / `ImageProcessingService` calls are guarded by try/catch or return-null patterns.
- In the catch block (or where a null result is handled after compression), add a snackbar:
  ```dart
  // Example — in the image pick flow:
  try {
    final bytes = await ImageProcessingService().compress(file);
    // ... proceed
  } catch (_) {
    if (mounted) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text(
          _localeProvider?.translate('image_load_failed')
              ?? "Couldn't load the image. Please try again.",
        )),
      );
    }
  }
  ```
- Check BOTH the camera path and the gallery path — each has its own flow.
- Also check if `ChatProvider` has any image-related catch blocks that currently swallow errors silently; if so, surface them the same way.

## Technical Details

- **`translate(key)` usage:** Task 012 calls `_localeProvider?.translate('voice_didnt_catch_that')` with a hardcoded English fallback. After this task lands, the translate call will return the proper localized string. The fallback is a safety net if the key is missing (it will not be missing after this task, but good defensive coding).
- **Where the voice string is consumed:** Task 012 (`ChatProvider._handleStream`, `onDone`) reads this key from `_localeProvider` and sets it as the assistant message content. This task only defines the string — consumption is in 012.
- **Where the image string is consumed:** This task also surfaces it. The `image_load_failed` key may additionally be referenced in future tasks (e.g., if a proper image error widget is added) — keep the key stable.
- **String tone:** Both strings are friendly, first-person, and action-oriented. Do not change tone or wording without product approval.
- **No new files needed:** Only `l10n_config.dart` and the existing chat tab file for the snackbar wiring.
