# Task 010: Chat composer mic icon + recording bar UI

## Context

The chat screen currently has a camera icon that lets users attach images. This task adds a mic icon next to it and, when recording is active, replaces the entire text-input area with a compact recording bar showing elapsed time and cancel/stop controls — similar to Messenger's voice-note UI. The recording bar must feel instant (NFR-2: <300 ms from tap to visible) and must be disabled when the AI is streaming a response (same rule as the camera icon today).

This task only handles UI and recording lifecycle (start/stop/cancel). Sending the resulting bytes to the provider is wired in task 012 (ChatProvider); this task calls `ChatProvider.sendMessage(audioBytes: ...)` once bytes are ready.

## Acceptance Criteria

- [ ] **FR-1 / Normal state:** Mic icon is visible next to the camera icon when `isStreaming == false`; has `Semantics(label: 'Record voice message')` for accessibility.
- [ ] **FR-1 / Streaming disabled:** When `isStreaming == true`, both the mic and camera icons are visually greyed out and `onTap` is null (not just visually disabled).
- [ ] **FR-2 / Tap-to-record:** Single tap on mic → `_isRecording` becomes true → composer area replaces with recording bar (`AnimatedSwitcher`) within 300 ms.
- [ ] **FR-2 / Recording bar contents:** Recording bar shows: animated pulse icon (driven by `amplitudeStream`), elapsed timer label in `M:SS` format (driven by `elapsedStream`), cancel button (×) on the left, stop-and-send button (■) on the right.
- [ ] **FR-2 / Auto-stop UI:** When `AudioRecordingService.onAutoStopped` fires, the recording bar disappears and the composer returns to default state (send is triggered automatically).
- [ ] **FR-2 / Stop-and-send:** Tapping ■ stops recording, gets bytes, calls `context.read<ChatProvider>().sendMessage('', audioBytes: bytes)`, returns composer to default state.
- [ ] **FR-3 / Cancel:** Tapping × calls `AudioRecordingService().cancel()` and returns composer to default state; no message is sent.
- [ ] **NFR-2 / Latency:** Recording bar is visible within 300 ms of mic tap (manual timing).
- [ ] **Permission — first use:** If `hasPermission() == false`, show a rationale dialog before requesting; if user denies, show snackbar "Microphone access is needed to record voice notes."; mic icon stays tappable (so user can retry via Settings).

### Step 3: Implement mic tap handler

```dart
Future<void> _onMicTap() async {
  // Permission check
  if (!await _audioService.hasPermission()) {
    // Show rationale dialog
    final granted = await showDialog<bool>(
      context: context,
      builder: (_) => AlertDialog(
        title: const Text('Microphone Access'),
        content: const Text('WalletAI needs microphone access to record voice expense notes.'),
        actions: [
          TextButton(onPressed: () => Navigator.pop(context, false), child: const Text('Not now')),
          TextButton(onPressed: () => Navigator.pop(context, true), child: const Text('Allow')),
        ],
      ),
    ) ?? false;
    if (!granted) {
      ScaffoldMessenger.of(context).showSnackBar(
        const SnackBar(content: Text('Microphone access is needed to record voice notes.')),
      );
      return;
    }
    // Request permission
    if (!await _audioService.requestPermission()) {
      ScaffoldMessenger.of(context).showSnackBar(
        const SnackBar(content: Text('Microphone access is needed to record voice notes.')),
      );
      return;
    }
  }
  // Start recording
  await _audioService.start();
  setState(() => _isRecording = true);
}
```

### Step 4: Implement _stopAndSend and _cancelRecording

```dart
Future<void> _stopAndSend() async {
  final bytes = await _audioService.stop();
  setState(() => _isRecording = false);
  if (bytes != null && bytes.isNotEmpty) {
    await context.read<ChatProvider>().sendMessage('', audioBytes: bytes);
  }
}

Future<void> _cancelRecording() async {
  await _audioService.cancel();
  setState(() => _isRecording = false);
}

  // bytes already captured inside the service's auto-stop; re-stop to get them

  // Implement per whichever pattern 002 exposes.
}
```

### Step 5: Add _RecordingBar widget and AnimatedSwitcher

- Replace the text-input row with an `AnimatedSwitcher`:

  - Shows × button (left) wired to `onCancel`, ■ button (right) wired to `onStop`.

## Interface Contract

### Receives from 002:
- File: `lib/services/audio_recording_service.dart`
  - `hasPermission() → Future<bool>`
  - `requestPermission() → Future<bool>`
  - `start() → Future<void>`
  - `stop() → Future<Uint8List?>`
  - `cancel() → Future<void>`
  - `elapsedStream → Stream<Duration>`
  - `amplitudeStream → Stream<Amplitude>`
  - `onAutoStopped: void Function(Uint8List?)? ` (or `VoidCallback?` — see note above)
- Guaranteed: After `start()` returns, `elapsedStream` emits within 100 ms.

### Produces for 012 (ChatProvider):
- Calls `ChatProvider.sendMessage('', audioBytes: bytes)` when stop-and-send or auto-stop fires.
- (Task 012 must extend `sendMessage` to accept `audioBytes` — this task depends on that extension being present before merging.)

## Technical Details

- **Approach (AD-6):** In-place `AnimatedSwitcher` swap, not a modal. Keeps the same height as the normal composer to prevent layout reflow causing jitter.
- **Streaming guard:** Read `ChatProvider.isStreaming` via `context.watch`; disable mic icon when true (same pattern as the existing camera button at line ~107).
- **`_isRecording` vs provider state:** Keep `_isRecording` as local widget state — it's purely UI state. The audio bytes are held by the service until handed to the provider on send.
- **Do NOT import `record` directly** in the tab — all `record` types come through `AudioRecordingService`.
- **Edge case — double tap:** Guard `_onMicTap` with `if (_isRecording) return;` to prevent a second recording starting before the first stops.
- **Edge case — navigation while recording:** `dispose()` cancels the recording so no dangling resources.
