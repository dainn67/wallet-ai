# Task 002: Implement AudioRecordingService singleton

## Context

WalletAI needs an audio recorder that starts on a single tap, stops on another, auto-stops at 30 seconds, and returns raw audio bytes that the provider can base64-encode and attach to the chat request. This service is the sole owner of the `record` package — no other file in the app imports it directly. It follows the same singleton pattern as `ImagePickerService` and `ImageProcessingService`, isolating the third-party package behind a thin facade so a package swap in the future is a one-file change.

## Description

Create `lib/services/audio_recording_service.dart` as a singleton following the existing `ImagePickerService` pattern. Expose a clean interface for start, stop, cancel, permission check, elapsed time, and amplitude. Enforce a hard 30-second cap via an internal `Timer`. Encode audio as AAC-LC 128 kbps mono (AD-4 from the epic). Export from `lib/services/services.dart`. Write unit tests covering the happy path and edge cases.

## Acceptance Criteria

- [ ] **FR-2 / Happy path:** `start()` begins recording within 300 ms; `stop()` returns a non-null `Uint8List` of audio bytes; bytes can be decoded as valid AAC audio.
- [ ] **FR-2 / Auto-stop at 30 s:** After 30 seconds without a manual `stop()`, the service auto-stops, fires the completion callback, and `stop()` / `cancel()` become no-ops until the next `start()`.
- [ ] **FR-3 / Cancel:** `cancel()` discards audio in progress and returns `Future<void>` without bytes; subsequent `stop()` returns null.
- [ ] **NFR-1 / File size:** `stop()` result for a 30-second recording is ≤ 500 KB raw (≈ 700 KB after base64 — well under the 5 MB PRD cap). Assert in unit test by checking bytes.length ≤ 524288.
- [ ] **NFR-2 / Latency:** `start()` must be synchronous enough that the caller observes the `elapsedStream` emitting within 300 ms of the call on a real device (manual verification).
- [ ] **Singleton:** `AudioRecordingService() == AudioRecordingService()` (same instance).
- [ ] **Mock support:** `setMockInstance` allows injecting a test double.

## Implementation Steps

### Step 1: Create lib/services/audio_recording_service.dart

```dart
import 'dart:async';
import 'dart:io';
import 'dart:typed_data';

import 'package:flutter/foundation.dart';
import 'package:path_provider/path_provider.dart';
import 'package:record/record.dart';
```

- Class: `AudioRecordingService` with static `_instance`, private `_internal()`, factory.
- Add `@visibleForTesting static void setMockInstance(AudioRecordingService? instance)`.

Internal state:
- `final AudioRecorder _recorder = AudioRecorder();` — one recorder instance, reused.
- `String? _tmpPath` — path of the in-progress recording.
- `bool _recording = false`.
- `Timer? _autoStopTimer` — fires after 30 s.
- `StreamController<Duration> _elapsedController` — broadcast, 100 ms ticks.
- `Timer? _elapsedTimer`.
- `VoidCallback? onAutoStopped` — caller sets this before `start()`; the service calls it when the 30-s timer fires (provider listens to send the message automatically).

Public API:

**`Future<bool> hasPermission()`**
- Call `await _recorder.hasPermission()`.
- Return the bool.

**`Future<void> start()`**
- Assert `!_recording`.
- Compute tmp path: `final dir = await getTemporaryDirectory(); _tmpPath = '${dir.path}/voice_${DateTime.now().millisecondsSinceEpoch}.m4a';`
- Call `await _recorder.start(RecordConfig(encoder: AudioEncoder.aacLc, bitRate: 128000, sampleRate: 44100, numChannels: 1), path: _tmpPath!);`
- Set `_recording = true`.
- Start elapsed timer: 100 ms periodic tick, increments `Duration` and adds to `_elapsedController`.
- Start auto-stop timer: `_autoStopTimer = Timer(const Duration(seconds: 30), _autoStop);`

**`Stream<Duration> get elapsedStream`** — `_elapsedController.stream`

**`Stream<Amplitude> get amplitudeStream`** — `_recorder.onAmplitudeChanged(const Duration(milliseconds: 100))`

**`Future<Uint8List?> stop()`**
- If `!_recording`, return null.
- Cancel timers (`_autoStopTimer?.cancel()`, `_elapsedTimer?.cancel()`).
- Call `await _recorder.stop()`.
- Set `_recording = false`.
- Read bytes: `final bytes = await File(_tmpPath!).readAsBytes();`
- Delete tmp file: `await File(_tmpPath!).delete();`
- `_tmpPath = null`.
- Return `Uint8List.fromList(bytes)`.

**`Future<void> cancel()`**
- If `!_recording`, return.
- Cancel timers.
- Call `await _recorder.stop()`.
- Set `_recording = false`.
- Delete tmp file if exists: `try { await File(_tmpPath!).delete(); } catch (_) {}`
- `_tmpPath = null`.

**Private `void _autoStop()`**
- Call `stop()` (stores future, fires callback).
- Call `onAutoStopped?.call()`.

  - `hasPermission() → Future<bool>`
  - `start() → Future<void>` — begins recording; sets `onAutoStopped` callback before calling
  - `stop() → Future<Uint8List?>` — returns AAC bytes or null if not recording
  - `cancel() → Future<void>` — discards in-progress recording
  - `elapsedStream → Stream<Duration>` — emits every 100 ms while recording
  - `amplitudeStream → Stream<Amplitude>` — emits mic amplitude while recording
  - `onAutoStopped: VoidCallback?` — set by caller; fired when 30-s cap is reached

### Produces for 012 (ChatProvider):
- `stop() → Future<Uint8List?>` — provider passes result to `sendMessage(audioBytes: …)`.

## Technical Details

- **Pattern:** Follow `lib/services/image_picker_service.dart` for singleton shape and `@visibleForTesting setMockInstance`.
- **Encoder config (AD-4):** AAC-LC (`AudioEncoder.aacLc`), 128 kbps bit rate, 44100 Hz, 1 channel. This yields ≈ 480 KB for 30 s — within NFR-1's 5 MB cap.
- **Temp file path:** Use `getTemporaryDirectory()` from `path_provider` (already in pubspec via existing image pipeline). File name includes epoch ms to avoid collision if start is called rapidly.
- **Amplitude stream:** `record ^5.x` exposes `AudioRecorder.onAmplitudeChanged(interval)` which returns `Stream<Amplitude>`. The UI uses `amplitude.current` (dBFS). Return the stream directly — do not buffer.
- **Edge case — audio session interruption (iOS):** An incoming call suspends the audio session. `record` may emit an error or stop silently. Wrap `_recorder.start(...)` and `_recorder.stop()` in try/catch; on catch, call `cancel()` to clean up state.
- **Edge case — permission denied:** `start()` will throw if called without permission. The UI (task 010) must check `hasPermission()` first and only call `start()` after the user grants. Do not add a permission check inside `start()` itself — keep the service thin.
