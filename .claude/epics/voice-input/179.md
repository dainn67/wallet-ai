# Task 001: Add record + permission_handler dependencies and platform plumbing

## Context

Voice input requires two capabilities the project currently lacks: (1) audio recording via a Flutter package, and (2) explicit microphone permission handling on iOS and Android. This task lays the entire dependency foundation — every subsequent task depends on these being in place. Without it, AudioRecordingService (task 002) cannot import the recording library and the app cannot request mic access.

## Description

Add `record: ^5.x` and `permission_handler: ^11.x` to `pubspec.yaml`. Add the required privacy strings and permissions to both native platform manifests. Run `fvm flutter pub get` and verify the project still builds on both platforms with no conflicts. No Dart code is written in this task — it is pure configuration.

Architecture note: The project uses FVM for Flutter version management; all Flutter/Dart commands must be prefixed with `fvm`.

## Acceptance Criteria

- [ ] **FR-1 / Prerequisite:** `fvm flutter pub get` exits 0 and both `record` and `permission_handler` appear in `pubspec.lock`.
- [ ] **FR-2 / Prerequisite (iOS):** First mic-tap on an iOS device/simulator surfaces a permission dialog containing the `NSMicrophoneUsageDescription` string (no crash, no silent denial).
- [ ] **FR-2 / Prerequisite (Android):** App includes `RECORD_AUDIO` permission in its manifest; `adb shell dumpsys package <package> | grep RECORD_AUDIO` lists the permission.
- [ ] **Build:** `fvm flutter build apk --debug` and `fvm flutter build ios --no-codesign` both succeed.

## Implementation Steps

### Step 1: Add packages to pubspec.yaml
- Open `pubspec.yaml`.
- Under `dependencies:`, add:
  ```yaml
  record: ^5.0.0
  permission_handler: ^11.0.0
  ```
- Keep alphabetical order with the surrounding entries.
- Run `fvm flutter pub get` and confirm resolution succeeds.

### Step 2: iOS — add NSMicrophoneUsageDescription
- Open `ios/Runner/Info.plist`.
- Add the following key-value pair inside the root `<dict>`:
  ```xml
  <key>NSMicrophoneUsageDescription</key>
  <string>WalletAI needs microphone access to record voice expense notes.</string>
  ```
- This string appears in the iOS system permission dialog. It must be present or iOS 14+ will crash on mic-permission request.
- If `NSMicrophoneUsageDescription` already exists in the file (unlikely), update its value to match the above.

### Step 3: Android — add RECORD_AUDIO permission
- Open `android/app/src/main/AndroidManifest.xml`.
- Add before the `<application>` tag:
  ```xml
  <uses-permission android:name="android.permission.RECORD_AUDIO"/>
  ```
- If it already exists, no change needed.

### Step 4: Verify build on both platforms
- Run `fvm flutter pub get`.
- Run `fvm flutter build apk --debug` — must exit 0.
- Run `fvm flutter build ios --no-codesign` — must exit 0.
- If any build fails, resolve the conflict (likely a transitive dependency version) before marking done.

## Technical Details

- **Approach:** Pure configuration — no Dart source changes.
- **Package rationale (AD-3):** `record ^5.x` is chosen over `flutter_sound` because it has a single responsibility (recording only), lighter native footprint, and built-in AAC encoding support on both iOS and Android. `permission_handler ^11.x` provides a cross-platform abstraction over the native permission APIs.
- **iOS privacy manifest note:** Newer Xcode / App Store requirements may require a `PrivacyInfo.xcprivacy` file listing `NSPrivacyAccessedAPIMicrophone`. Check the `record` package's README for its current guidance; add the file if required to pass App Store review.
- **Edge case:** If `permission_handler`'s transitive dependencies conflict with any existing package (check `flutter pub get` output), pin to the highest compatible version that satisfies all constraints.
