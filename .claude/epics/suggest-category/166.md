### Step 3: Write `docs/features/suggest-category.md`

Create a new doc following the format of existing files in `docs/features/` (check one existing file for structure):

```markdown
# Suggest Category

## Overview
When a user logs an expense the AI cannot match to an existing category, the server returns a `suggested_category` object per record. The Flutter client parses this and displays an inline suggestion banner below the unclassified record card in the chat.

## User Flow
1. User sends message: "50k tài khoản Netflix"
2. Server returns record with `category_id: "-1"` and `suggested_category: {name: "Streaming", type: "expense", parent_id: -1, message: "..."}`
3. After stream completes, `ChatProvider._handleStream` parses suggestion into `Record.suggestedCategory`
4. Chat bubble renders `RecordWidget` + `SuggestionBanner` beneath it
5. User taps Confirm → `RecordProvider.resolveCategoryByNameOrCreate` → category created or reused → record updated in DB → banner disappears
6. Or: User taps Cancel → `chatProvider.updateMessageRecord` clears suggestion → banner disappears, record stays Uncategorized

## Technical Notes
- `SuggestedCategory` is a transient model field — NOT persisted to SQLite
- Server response format: `{"name": "...", "type": "expense"|"income", "parent_id": -1 or existing_id, "message": "..."}`
- Duplicate guard: if a category with the same name+parentId already exists, it is reused (no duplicate created)
- Session-only: app restart clears suggestion banners (records with `categoryId: -1` remain accessible via Records tab)

## Key Files
- `lib/models/suggested_category.dart` — Data class + defensive fromJson
- `lib/models/record.dart` — Transient `suggestedCategory` field
- `lib/providers/chat_provider.dart` — Parses suggestion in `_handleStream`
- `lib/providers/record_provider.dart` — `resolveCategoryByNameOrCreate` helper
- `lib/components/suggestion_banner.dart` — UI widget
- `lib/components/chat_bubble.dart` — Integration point (line ~78)
```
