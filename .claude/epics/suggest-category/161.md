# Task: SuggestedCategory model + transient field on Record

## Context

When the AI cannot classify a user's expense, the server includes a `suggested_category` object per record in the JSON response (alongside `category_id: "-1"`). Currently the Flutter client ignores this entirely тАФ the record is saved as Uncategorized and the suggestion is discarded. This task creates the data types that carry the suggestion through the app: a new `SuggestedCategory` data class and a transient (never-persisted) nullable field on the existing `Record` model.

## Description

Create `lib/models/suggested_category.dart` with a simple data class and a defensive `fromJson` factory that returns `null` on any malformed input. Extend `lib/models/record.dart` to carry a nullable `suggestedCategory` field тАФ this field is intentionally excluded from `toMap()` and `fromMap()` (never written to or read from SQLite). Thread the field through `Record`'s constructor and `copyWith`, using a `bool clearSuggestedCategory = false` flag to handle nullable reset (standard Dart `??` idiom cannot distinguish "don't change" from "set to null"). Export the new class from the barrel file.

Architecture decision AD-1: transient on Record rather than a parallel provider map.
Architecture decision AD-4: `bool clearSuggestedCategory` flag for nullable-reset in copyWith.

## Acceptance Criteria

- [ ] **FR-2 / Happy path:** `SuggestedCategory.fromJson({'name': 'Streaming', 'type': 'expense', 'parent_id': -1, 'message': 'Create it?'})` returns a non-null instance with all fields populated correctly.
- [ ] **FR-2 / Model roundtrip:** A `Record` constructed with a non-null `suggestedCategory` тАФ when `toMap()` is called тАФ does NOT include any `suggested_category` key in the result map.
- [ ] **FR-2 / fromMap roundtrip:** A `Record` reconstructed via `Record.fromMap(map)` always has `suggestedCategory == null` regardless of what is in the map.
- [ ] **FR-2 / copyWith clear:** `record.copyWith(clearSuggestedCategory: true)` returns a new Record with `suggestedCategory == null`, even if no other fields changed.
- [ ] **FR-2 / copyWith passthrough:** `record.copyWith()` (no args) on a record with non-null `suggestedCategory` returns a copy that still has the same `suggestedCategory` reference.
- [ ] **NFR-2 / Missing name:** `SuggestedCategory.fromJson({'type': 'expense', 'parent_id': -1, 'message': 'x'})` (no `name` key) returns `null`.
- [ ] **NFR-2 / Invalid type:** `SuggestedCategory.fromJson({'name': 'X', 'type': 'other', 'parent_id': -1, 'message': 'x'})` returns `null`.
- [ ] **NFR-2 / Empty name:** `SuggestedCategory.fromJson({'name': '', 'type': 'expense', 'parent_id': -1, 'message': 'x'})` returns `null`.
- [ ] **NFR-2 / Null input:** `SuggestedCategory.fromJson(null)` returns `null` without throwing.
- [ ] **NFR-2 / Non-map input:** `SuggestedCategory.fromJson('bad string')` returns `null` without throwing.

## Implementation Steps

### Step 1: Create `lib/models/suggested_category.dart`

- Create the file with a `SuggestedCategory` data class:
  ```dart
  class SuggestedCategory {
    final String name;
    final String type;    // 'expense' or 'income'
    final int parentId;   // -1 for top-level
    final String message;
  }
  ```
- Add a `const` constructor.
- Add a static `SuggestedCategory? fromJson(dynamic json)` factory:
  - Wrap entire body in `try { ... } catch (_) { return null; }`
  - If `json` is not a `Map<String, dynamic>` тАФ return `null`
  - Extract `name` (String), validate non-empty тАФ if null or empty, return `null`
  - Extract `type` (String), validate it's `'expense'` or `'income'` тАФ if invalid, return `null`
  - Extract `parentId` as int: `(json['parent_id'] as num?)?.toInt() ?? -1`
  - Extract `message` as String: `json['message'] as String? ?? ''`
  - Return `SuggestedCategory(name: name, type: type, parentId: parentId, message: message)`

### Step 2: Extend `lib/models/record.dart`

- Read the current `record.dart` fully before editing to understand the field order and `copyWith` signature.
- Add `final SuggestedCategory? suggestedCategory;` field to the class (import `suggested_category.dart`).
- Add `this.suggestedCategory` as an optional named parameter in the constructor (default `null`).
- In `toMap()`: do NOT add `suggestedCategory` тАФ leave the method entirely unchanged.
- In `fromMap()`: do NOT read `suggestedCategory` тАФ leave the method entirely unchanged.
- In `copyWith()`:
  - Add parameter `SuggestedCategory? suggestedCategory` (mirrors existing nullable pattern).
  - Add parameter `bool clearSuggestedCategory = false`.
  - For the field in the return expression: `suggestedCategory: clearSuggestedCategory ? null : (suggestedCategory ?? this.suggestedCategory)`
  - All other fields follow the existing `param ?? this.field` pattern unchanged.
- Grep for all `.copyWith(` usages on `Record` across the codebase to confirm no existing caller is broken (they pass no `suggestedCategory` arg, so they're unaffected).

### Step 3: Export from barrel

- Add `export 'suggested_category.dart';` to `lib/models/models.dart` (follow the existing export order).
