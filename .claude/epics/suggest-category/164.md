# Task: Build SuggestionBanner widget

## Context

When the AI cannot classify an expense, the server returns a suggestion (`name`, `type`, `parentId`, `message`). This task creates the widget that surfaces this suggestion to the user inline in the chat — a banner card appearing beneath the unclassified record card, showing the AI's message, the proposed category name, type badge, and two action buttons (Confirm / Cancel). The banner operates via callbacks so it can be tested without provider context.

## Interface Contract

### Receives from T001:
- File: `lib/models/suggested_category.dart`
  - Type: `SuggestedCategory` with `name: String`, `type: String`, `parentId: int`, `message: String`
- File: `lib/models/record.dart`
  - Type: `Record` passed to `SuggestionBanner` constructor (needed so T020 can call `chatProvider.updateMessageRecord` with the updated record)

### Produces for T020 (Integration):
- File: `lib/components/suggestion_banner.dart`
  - Widget: `SuggestionBanner({required Record record, required String messageId, required SuggestedCategory suggestion, required Future<void> Function() onConfirm, required VoidCallback onCancel})`
  - Guaranteed: Confirm button is disabled during async `onConfirm` call (prevents double-fire); re-enabled on error.

## Acceptance Criteria

- [ ] **FR-3 / Renders correctly:** Banner shows `suggestion.message` text, the category name (`suggestion.name`), and a type badge (`suggestion.type`). Confirm and Cancel buttons are visible.
- [ ] **FR-3 / Type badge color:** Badge is visually distinct for `expense` (red/orange) vs `income` (green), matching existing `RecordWidget` type indicator style.
- [ ] **NFR-3 / Double-tap guard:** Tapping Confirm twice rapidly fires `onConfirm` only once (button is disabled after first tap until `onConfirm` completes or errors).
- [ ] **NFR-3 / Re-enable on error:** If `onConfirm` throws or returns exceptionally, the Confirm button becomes enabled again.
- [ ] **FR-3 / Cancel calls callback:** Tapping Cancel fires `onCancel` immediately (synchronous, no loading state needed).
- [ ] **FR-3 / Callback-driven:** Widget does NOT directly access `RecordProvider` or `ChatProvider` via `context.read` — all side effects are delegated to `onConfirm` and `onCancel` callbacks.

## Implementation Steps

### Step 1: Create `lib/components/suggestion_banner.dart`

Create a `StatefulWidget` named `SuggestionBanner` with:

**Constructor parameters:**
```dart
const SuggestionBanner({
  super.key,
  required this.record,
  required this.messageId,
  required this.suggestion,
  required this.onConfirm,   // Future<void> Function()
  required this.onCancel,    // VoidCallback
});
```

**State fields:**
```dart
bool _isProcessing = false;
```

**Build method layout** (match visual style of `RecordWidget` — same horizontal padding, same border radius):
- Outer: `Padding` with left/right padding matching `RecordWidget`
- `Card` with `BorderRadius.circular(12)` and a subtle left border or background tint to visually associate it with the record above
- Column children:
  1. AI message text: `Text(widget.suggestion.message, style: ...bodyMedium...)` — muted/secondary color
  2. Row: category name chip + type badge
     - Chip: `Container` with rounded border, label `widget.suggestion.name`
     - Type badge: small pill, red (`Colors.red.shade100`) for `'expense'`, green (`Colors.green.shade100`) for `'income'`, text `widget.suggestion.type`
  3. Row (right-aligned): Cancel button (TextButton, secondary), Confirm button (FilledButton or ElevatedButton, primary)

**Confirm tap handler:**
```dart
Future<void> _handleConfirm() async {
  if (_isProcessing) return;
  setState(() => _isProcessing = true);
  try {
    await widget.onConfirm();
  } catch (_) {
    setState(() => _isProcessing = false);
  }
}
```

**Cancel tap handler:**
```dart
void _handleCancel() => widget.onCancel();
```

**Confirm button:**
```dart
FilledButton(
  onPressed: _isProcessing ? null : _handleConfirm,
  child: _isProcessing
      ? const SizedBox(width: 16, height: 16, child: CircularProgressIndicator(strokeWidth: 2))
      : const Text('Confirm'),
)
```

**Cancel button:**
```dart
TextButton(
  onPressed: _handleCancel,
  child: const Text('Cancel'),
)
```

### Step 2: Match visual style to existing components

- Read `lib/components/record_widget.dart` (or wherever `RecordWidget` is defined) to get: horizontal padding value, card border radius, color tokens used for expense/income.
- Use the same padding and border-radius values to ensure the banner visually hugs the record card above it.
- Do NOT hardcode colors — use the existing theme/color pattern (e.g., if `RecordWidget` uses `Theme.of(context).colorScheme.error` for expense, use the same).

## Technical Details

- **Approach:** Stateful only for `_isProcessing` flag. All business logic in callbacks (AD-3).
- **Files to create:** `lib/components/suggestion_banner.dart`
- **Patterns to follow:** `lib/components/record_widget.dart` for visual style. `lib/components/popups/category_form_dialog.dart` for button/loading pattern.
- **Edge cases:**
  - `suggestion.message` may be long — use `softWrap: true` on the Text widget.
  - `suggestion.name` may be long — chip should ellipsize or allow wrapping.
  - Do NOT access providers directly in this widget — the integration task (T020) wires the callbacks.
