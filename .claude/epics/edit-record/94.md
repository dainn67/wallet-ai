# Task: Implement Popup Logic and Validation

## Context
Inputs must be validated before returning the updated record.

## Description
Add validation logic to `EditRecordPopup` and ensure it returns an updated `Record` object when "Save" is pressed.

## Implementation Steps
1. Open `lib/components/popups/edit_record_popup.dart`.
2. Add validation for Amount: must be a valid positive double.
3. Add validation for Description: must not be empty.
4. Ensure Money Source and Category selectors are functional.
5. On "Save", construct a new `Record` object using `record.copyWith(...)` and call `Navigator.pop(context, updatedRecord)`.

## Acceptance Criteria
- [ ] Error messages appear for empty description or invalid amount.
- [ ] Popup returns a `Record` object with updated values.
- [ ] Clicking "Cancel" returns `null`.
