# Task: Update RecordWidget and Integration

## Context
The "Edit" entry point needs to be added to the UI and wired to the update logic.

## Description
Modify `RecordWidget` to include an edit button and integrate it with the `EditRecordPopup` in the `RecordsTab`.

## Implementation Steps
1. Open `lib/components/record_widget.dart`.
2. Add a final `bool isEditable` field (default to `false`) to the class and constructor.
3. In the `build` method, if `isEditable` is true, add a trailing `IconButton` with `Icons.edit_rounded`.
4. The `IconButton`'s `onPressed` should call a callback or handle the popup logic.
5. Open `lib/screens/home/tabs/records_tab.dart`.
6. Update the record list to pass `isEditable: true` to `RecordWidget`.
7. Implement the `onEdit` logic:
    - Show `EditRecordPopup`.
    - If a new record is returned, call `context.read<RecordProvider>().updateRecord(updatedRecord)`.

## Acceptance Criteria
- [ ] Edit button is visible on records in the Records tab.
- [ ] Edit button is NOT visible on records in other parts of the app (e.g., Chat tab).
- [ ] Tapping the edit button opens the popup.
- [ ] Saving from the popup updates the UI immediately.
