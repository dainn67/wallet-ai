# Task: Verification and Cleanup

## Context
Final quality gate to ensure data integrity and UI consistency.

## Description
Run manual and automated tests to verify the record editing flow and balance calculations.

## Implementation Steps
1. Perform a fresh app install or clear data to trigger DB v6 initialization.
2. Add multiple sources and records.
3. Edit a record's amount and verify both the source balance and total balance update correctly.
4. Edit a record's source and verify that the old source balance is restored and the new source balance is adjusted.
5. Verify that the `last_updated` field is correctly set.
6. Run all existing regression tests.

## Acceptance Criteria
- [ ] End-to-end edit flow works without errors.
- [ ] Wallet balances are 100% accurate after multiple edits.
- [ ] No regressions in existing functionality.
