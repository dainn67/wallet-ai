# Task: Implement dynamic environment switching in AppConfig

## Context
The app currently has a hardcoded `AppEnvironment.dev`. We need a way to switch between `dev` and `prod` at build time without changing code, ensuring security and flexibility.

## Description
Refine the `AppConfig` singleton to use `const String.fromEnvironment('ENVIRONMENT')` to determine the active environment. This allows us to set the environment during the build process (e.g., `flutter run --dart-define=ENVIRONMENT=prod`).

## Acceptance Criteria
- [ ] **FR-1 / Environment Switch:** Build with `--dart-define=ENVIRONMENT=prod` -> `AppConfig().environment` returns `AppEnvironment.prod`.
- [ ] **FR-1 / Default Environment:** Build without flag -> `AppConfig().environment` defaults to `AppEnvironment.dev`.
- [ ] **NFR-1 / Security:** Base URLs for `dev` and `prod` are correctly returned based on the environment.

## Implementation Steps

### Step 1: Update AppConfig environment initialization
- Modify `lib/config/app_config.dart`
- Change `environment` property to be initialized based on `const String.fromEnvironment('ENVIRONMENT')`.
- Ensure it defaults to `dev` if the string is empty or invalid.

### Step 2: Refine baseUrl and timeout logic
- Ensure `baseUrl` getter correctly uses the dynamic `environment`.
- Verify `connectTimeout` and `receiveTimeout` are appropriate for both environments.

## Technical Details
- **Approach:** Use Flutter's built-in environment variable support for compile-time configuration.
- **Files to modify:** `lib/config/app_config.dart`.
- **Patterns to follow:** Singleton pattern already implemented in the file.
- **Edge cases:** Handle invalid strings in `fromEnvironment` by defaulting to `dev`.
